# 专利 P2：面向TCC的可重构原语硬件IP核阵列
# Patent P2: Reconfigurable Primitive IP Core Array for TCC
# 状态：📋 框架完成 | 目标提交：2026年7月（CNIPA）
---

## 核心创新点

单一计算节点集成4个可独立工作的计算引擎（GEMM脉动阵列 + FOLD归约树 + MAPS逐元素+LUT + SCAN前缀树），通过统一互连端口接入SDI可编程网络，构成TCC-11硬件基础单元。

---

## 权利要求摘要

### 权利要求 1（独立——计算节点）

一种拓扑中心计算节点，包含以下功能模块：

- **矩阵乘加单元（GEMM Engine）**：M×K乘加器脉动阵列，支持INT4/INT8/FP16/BF16，累加器≥32位
- **向量归约单元（FOLD Engine）**：log₂M级归约树，支持SUM/MAX/MIN/ARGMAX/ARGMIN/PROD运行时切换
- **逐元素映射单元（MAPS Engine）**：M个并行可配置功能单元（ALU + 可编程LUT深度≥256 + 位操作单元）
- **前缀扫描单元（SCAN Engine）**：2·log₂M级前缀树（Brent-Kung或Kogge-Stone），inclusive/exclusive可配
- **本地SRAM**：容量≥64KB，双端口或多bank结构
- **互连端口**：≥4方向数据端口，通过可编程路由寄存器连接节点间互连网络

### 权利要求 2（GEMM规格）

M×K阵列中M≥8、K≥8；INT4模式下每周期2次乘加；累加器位宽≥32位。

### 权利要求 3（LUT动态重加载）

可编程LUT可在运行时通过tcc.LINK控制原语重新加载内容，加载延迟≤100个时钟周期，支持ReLU/GELU/SiLU/Sigmoid/exp/复数旋转因子动态切换。

### 权利要求 4（独立——阵列级）

一种拓扑中心计算阵列，包含N个权利要求1所述计算节点和一个SDI控制器：

- 可编程互连网络：N×N交叉开关 + 连接矩阵寄存器组
- SDI控制器：拓扑配置引擎 + 同步屏障逻辑 + Lamport逻辑时钟生成器
- N范围：4至65536；超单芯片时通过UCIe或自定义芯片间互连级联

## 稀疏拓扑支持扩展（新增权利要求 5–9，2026-04-30）

> 基于稀疏拓扑原语理论，补充P2专利的稀疏硬件支持权利要求

### 权利要求 5（从属——链路门控，稀疏物理层）

根据权利要求4所述的计算阵列，其特征在于，所述可编程互连网络的每条链路配备**激活门控单元**，包含：

（a）**门控寄存器**：1比特激活标志，可由SDI控制器或PRUNE原语指令写入；
当激活标志为0时，对应忆阻器单元切换至高阻态（≥1 MΩ），物理断开链路，链路静态功耗降至≤1%额定值；

（b）**掩码寄存器组**：N×N比特矩阵，存储当前拓扑的稀疏激活模式，支持按行/列批量写入，用于快速切换稀疏拓扑配置；

（c）**稀疏度寄存器**：记录当前激活链路比例 ρ（定点数，0.0–1.0），供Γst监测单元读取。

### 权利要求 6（从属——PRUNE原语硬件支持）

根据权利要求5所述的计算阵列，其特征在于，SDI控制器支持**PRUNE原语**指令：

```
PRUNE(mask_matrix[N×N])
```

执行语义：
1. 将mask_matrix写入掩码寄存器组；
2. 对mask=0的链路触发忆阻器高阻切换，切换时延≤10 ns；
3. 更新稀疏度寄存器 ρ = popcount(mask) / N²；
4. 生成TICK时间戳，广播拓扑变更事件至所有激活节点。

**技术效果**：95%稀疏度工作负载下，互连网络静态功耗降低≥90%。

### 权利要求 7（从属——PACK原语硬件支持，拓扑压缩）

根据权利要求5所述的计算阵列，其特征在于，SDI控制器支持**PACK原语**指令：

```
PACK(active_indices[K], virtual_id_map[K→K'])
```

执行语义：
1. 从N个节点中选取K个激活节点（K < N），建立虚拟ID映射表（K'×K'紧凑子拓扑）；
2. 将N-K个非激活节点置于低功耗待机态（时钟门控 + 链路断开）；
3. 后续原语调用在K'节点紧凑子拓扑上执行，逻辑正确性由ID映射表保证；
4. UNPACK原语恢复原始N节点拓扑和ID映射。

**压缩比**：拓扑规模从N降至K，计算与通信开销按比例降低；理论压缩上界为 N/K，由任务信息熵决定实际可达压缩比。

### 权利要求 8（从属——FOLD_S稀疏归约引擎）

根据权利要求1所述的计算节点，其特征在于，FOLD Engine支持**带掩码的稀疏归约模式（FOLD_S）**：

```
FOLD_S(data, mask, op)  → 仅对mask[i]=1的元素执行归约操作op
```

（a）**掩码寄存器**：与数据寄存器等位宽，由MAPS Engine写入；
（b）**选择器网络**：归约树每级节点前置1bit掩码选择器，mask=0的路径不参与计算；
（c）**稀疏感知调度**：连续零区间自动跳过（skip），减少归约树无效级联；
计算延迟从 log₂M 降至 log₂(nnz)，其中 nnz 为非零元素数。

### 权利要求 9（从属——Γst驱动的自动拓扑压缩）

根据权利要求4所述的计算阵列，其特征在于，SDI控制器包含**Γst监测单元**，实现自动稀疏压缩：

（a）**Γst估算器**：以时间窗口T（可配，默认256时钟周期）统计链路激活率和节点间信息流密度，按简化CST公式计算当前拓扑的 Γst 估算值；

（b）**阈值比较器**：将 Γst 与预置阈值 θ（默认 1/√2 ≈ 0.707）比较；

（c）**自动压缩触发器**：当 Γst < θ 持续时间超过 Δt（可配，默认1024周期），自动执行 PRUNE → PACK 操作序列，将拓扑压缩至最小有效子拓扑；当 Γst ≥ θ 时，触发 UNPACK 恢复全拓扑；

**技术效果**：无需软件干预，硬件自适应稀疏工作负载，在GNN/稀疏Transformer等动态稀疏场景下自动节能。

## 稀疏原语完整生命周期（权利要求5–9的协同工作）

```
┌─ 检测阶段 ──────────────────────────────────────┐
│  Γst监测单元评估当前拓扑激活率              │
│  若 Γst < 0.707 → 进入压缩流程             │
└────────────────────────────────────────────┘
         ↓
┌─ PRUNE ──────────────────────────────────────┐
│  按掩码断开非激活链路                       │
│  忆阻器→高阻态，功耗降低90%               │
└────────────────────────────────────────────┘
         ↓
┌─ PACK ───────────────────────────────────────┐
│  K个激活节点重映射到K'×K'紧凑子拓扑        │
│  N-K个节点进入待机态                       │
│  压缩比最高：N/K                           │
└────────────────────────────────────────────┘
         ↓
┌─ FOLD_S / FUSE_S / PULL_S ──────────────────┐
│  在压缩后的稀疏子拓扑上执行计算           │
│  延迟从 log₂N → log₂(nnz)               │
└────────────────────────────────────────────┘
         ↓
┌─ TICK + UNPACK ──────────────────────────────┐
│  同步状态快照，恢复原始N节点拓扑           │
└────────────────────────────────────────────┘
```
