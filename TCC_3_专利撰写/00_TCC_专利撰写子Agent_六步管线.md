# TCC 专利撰写子 Agent

> **角色定位：** 专门处理发明专利全流程的子 Agent，与论文撰写管线并行运行。
> **核心原则：** 每一步都产出可追溯的中间产物——专利检索报告、可专利性评估、交底书、权利要求树、附图、申请文件。
> **触发条件：** 用户提及"写专利"、"专利交底书"、"patent"、"权利要求"、"专利申请"时自动激活。

---

## 一、专利撰写管线：6 步可追溯流程

```
[1.现有技术检索] → [2.可专利性评估] → [3.交底书撰写] → [4.权利要求构建] → [5.附图绘制] → [6.申请文件交付]
       ↓                ↓                ↓                ↓                ↓              ↓
   检索报告         评估报告          交底书MD        权利要求树        正式附图       申请文件包
  (专利库+论文)   (新颖性+创造性)   (技术方案+实施例)  (独立+从属)      (Visio/PNG)   (说明书+权利要求+附图)
```

---

## 二、6 步工序详述

### 工序 P1：现有技术检索（Prior Art Search）

| 项目 | 内容 |
|------|------|
| **目标** | 全面检索与发明相关的现有专利、论文、公开技术 |
| **检索范围** | CNIPA（中国）、USPTO（美国）、WIPO（PCT）、EPO（欧洲）、Google Patents |
| **检索策略** | 关键词组合 + IPC 分类号 + 申请人追踪 + 前引/后引文献扩展 |
| **输出** | `P1_Prior_Art_Search_Report.md`：列出最相关的 10-20 件现有技术，每件标注公开号、相关性等级（A/X/Y）、关键区别 |
| **存储** | Zotero `TCC_专利` 文件夹 + `TCC_3_专利撰写/检索报告/` |
| **检查点** | 能否明确指出本发明与每件最相关现有技术的区别？ |

**TCC 当前状态：**
- R1（SDI架构）：⬜ 系统性检索未进行
- R2（网内归约）：⬜ 系统性检索未进行（重点关注 SHARP/In-Network Computing 专利族）

### 工序 P2：可专利性评估（Patentability Assessment）

| 项目 | 内容 |
|------|------|
| **目标** | 评估发明的新颖性（Novelty）、创造性（Inventive Step）和实用性（Industrial Applicability） |
| **输入** | 工序 P1 的检索报告 |
| **评估维度** | ① 新颖性：技术特征是否全部被单件现有技术公开？② 创造性：区别技术特征是否"显而易见"？③ 实用性：能否在产业上制造或使用？ |
| **输出** | `P2_Patentability_Assessment.md`：逐项分析 + 总体结论（建议申请/建议补充实验/不建议申请）+ 风险提示 |
| **存储** | `TCC_3_专利撰写/评估报告/` |
| **检查点** | "区别技术特征"是否足够具体（不是功能性描述，而是结构性限定）？ |

**TCC 当前状态：**
- R1：⬜ 可专利性评估未进行
- R2：⬜ 可专利性评估未进行

### 工序 P3：交底书撰写（Disclosure Drafting）

| 项目 | 内容 |
|------|------|
| **目标** | 撰写完整的专利交底书，作为正式申请文件的蓝本 |
| **输入** | 工序 P2 的评估报告 + 技术文档 |
| **章节结构** | ① 发明名称 ② 技术领域 ③ 背景技术（含现有技术缺陷） ④ 发明要解决的技术问题 ⑤ 技术方案（总体架构+核心方法） ⑥ 有益效果 ⑦ 附图说明 ⑧ 具体实施方式（≥2个实施例） |
| **输出** | `P3_Disclosure_[专利名].md` |
| **存储** | `TCC_3_专利撰写/交底书/` |
| **检查点** | "技术问题-技术方案-技术效果"三段论是否完整且逻辑自洽？实施例是否足够具体（"本领域技术人员能否根据描述再现"）？ |

**TCC 当前状态：**
- R1：✅ 完整交底书 v2 已完成（`专利A_*_完整交底书.md`，26.4 KB）
- R2：✅ 完整交底书 v2 已完成（`专利B_*_完整交底书.md`，28.4 KB）

### 工序 P4：权利要求构建（Claims Construction）

| 项目 | 内容 |
|------|------|
| **目标** | 构建多层次权利要求树，最大化保护范围 |
| **输入** | 工序 P3 的交底书 |
| **构建方法** | ① 提取技术方案的"必要技术特征"→独立权利要求 ② 对每个必要技术特征添加限定→从属权利要求 ③ 构建"系统+方法+装置"多维保护 |
| **输出** | `P4_Claims_Tree.md`：权利要求树（树形结构，独立权利要求为根，从属权利要求为枝）+ 每条权利要求的"被侵权检测"（如何判断他人是否侵权） |
| **存储** | `TCC_3_专利撰写/权利要求/` |
| **检查点** | 独立权利要求是否"最宽但有效"（不包含非必要技术特征）？从属权利要求是否有"退守层次"（最坏情况下仍有保护）？ |

**TCC 当前状态：**
- R1：✅ 10条权利要求草案已完成（2独立+8从属）
- R2：✅ 12条权利要求草案已完成（2独立+10从属）
- ⬜ 两件专利的"权利要求区分说明"待撰写（防止同日申请被驳回）

### 工序 P5：附图绘制（Patent Drawings）

| 项目 | 内容 |
|------|------|
| **目标** | 绘制符合专利法要求的正式附图 |
| **要求** | 黑白线条图、组件编号（1, 2, 3...）、无文字标注（仅在说明书中说明）、分辨率≥300 dpi |
| **工具** | Visio / AutoCAD / Python matplotlib / Inkscape → 导出为 PNG 或 PDF |
| **输出** | 4-6张正式附图 + 附图说明 |
| **存储** | `TCC_3_专利撰写/figures/` |
| **检查点** | 每张图是否自包含？组件编号是否与说明书完全一致？ |

**TCC 当前状态：**

- R2：⬜ 仅有 ASCII 草图，正式附图未绘制（需：系统架构图 + 流水线时序图 + 级联结构图 + 状态表流程图）

### 工序 P6：申请文件交付（Filing Package）

| 项目 | 内容 |
|------|------|
| **目标** | 组装完整的专利申请文件包 |
| **组成** | ① 发明专利请求书 ② 说明书 ③ 权利要求书 ④ 说明书附图 ⑤ 摘要 + 摘要附图 ⑥ 同日申请声明（如有） ⑦ 优先权证明文件（如有） |
| **输出** | `P6_Filing_Package/`：所有文件 PDF + 可编辑源文件 |
| **对接** | 专利代理人最终审核 + 递交 CNIPA |
| **检查点** | 所有文件是否满足 CNIPA 格式要求？同日申请声明是否准备？ |

## 四、子 Agent 使用指南

### 激活方式

在 Codex 对话中直接说：

```
@专利撰写 请对"面向网络中心计算的SDI系统架构"进行现有技术检索
@专利撰写 请评估R1交底书的可专利性
@专利撰写 请为R2构建权利要求树
@专利撰写 请绘制R1的系统总体框图
```

### 与论文管线的协同

```
            ┌──────────────────┐
            │   论文撰写管线   │ ← 8步工序（检索→交付）
            │   (Codex + ARS)  │
            └────────┬─────────┘
                     │ 共享
            ┌────────v─────────┐
            │  Zotero 知识库   │ ← 文献+证据+中间产物
            └────────┬─────────┘
                     │ 共享
            ┌────────v─────────┐
            │  专利撰写子Agent │ ← 6步工序（检索→递交）
            │  (独立管线)      │
            └──────────────────┘
```

- **共享检索结果**：论文的文献检索可复用为专利的现有技术背景
- **共享技术描述**：论文中的方法描述可直接转为专利的技术方案章节
- **保护边界区分**：论文公开 = 专利的现有技术；专利保护 = 论文中未公开的工程实现细节
