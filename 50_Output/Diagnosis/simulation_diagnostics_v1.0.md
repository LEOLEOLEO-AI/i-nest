# iNEST 仿真实验诊断报告
## 系统性漏洞、瑕疵与改进建议表

---

## 【执行摘要】

基于 v30 多物种仿真器和已发表结论的审视，识别出以下核心问题：

| 级别 | 问题数量 | 关键领域 | 风险等级 |
|-----|--------|--------|--------|
| 🔴 严重 | 3 | 统计学检验缺失、对照组缺失、基准不清 | 高 |
| 🟠 中等 | 5 | 参数正当性、物理约束、跨物种泛化性 | 中 |

**总计**：12 项问题需要系统改进

## 【问题诊断表】

### 类别 1: 统计学与结论佐证（🔴 严重）

#### 问题 1.1：缺少统计显著性检验

**现状**：
- v30 报告 5 个生物指标都"✅ PASS"
- 但无任何 p-value、置信区间、t-test 等统计量
- 例：σ = 4.238 vs 目标 ≥4.0，差异是否显著未知

**具体漏洞**：
```
指标         | 实测 | 目标    | PASS | 缺失的统计量
------------|------|--------|------|---------------
σ(平均度)   | 4.238| ≥4.0   | ✅   | p-value? 误差棒?
α(度相关)   | 2.099| 1.5-2.5| ✅   | 95% CI?
C(聚类系数) | 0.328| ≥0.30  | ✅   | 标准差?
L(路径长度) | 2.398| 2.0-3.5| ✅   | 与野生型的 t-test?
EL(电化学比)| 27.3%| 15-28% | ✅   | 比例的置信区间?
```

**为什么严重**：
- "达标"判断完全基于点估计，无可信区间
- 如果σ = 4.239，目标 = 4.240，是否应该"不达标"？
- 无法判断结果的鲁棒性

**改进建议**：
- [ ] 对每个指标计算 95% CI（通过自助法或理论分布）
- [ ] 针对 5 个指标进行多重比较校正（Bonferroni）
- [ ] 绘制指标的分布直方图 + 正态性检验（QQ 图）
- [ ] 与 C.elegans 野生型数据进行 t-test 对照

---

#### 问题 1.2：缺少对照组 (Control Group)

**现状**：
- v30 仿真结果都基于"理想条件"
  - 完美的初始拓扑（来自 WormAtlas）
  - 理想的学习规则（STDP、可塑性）
  - 无噪声、无扰动
  
**漏洞**：
```
缺失的对照条件：

1. 随机网络对照 (Null Model)
   - 保持度分布，随机重连
   - 是否与仿真结果有统计显著差异？
   
2. 传统 FPGA/标准神经网络对照
   - 相同输入，对比 MNIST 性能
   - 没有量化优势数据
   
3. 扰动稳定性对照
   - 移除 10%、20%、50% 的突触
   - 测试"鲁棒性"能否维持
   - 当前完全缺失

4. 不同初始拓扑对照
   - 不同 C.elegans 个体的差异
   - 仿真结果是否稳定？
```

**为什么严重**：
- 论文声称"临界态"，但未与随机网络对比
- 无法证明"结果来自 iNEST 机制"而非"输入数据的人工制品"

**改进建议**：
- [ ] 生成 3 种 Null model：
  - 配置模型（度分布相同，随机连接）
  - ER 随机图（相同节点数、边数）
  - Power-law 随机网络（相同 γ_s）
- [ ] 对照组与仿真组进行 KL 散度 / Wasserstein 距离计算
- [ ] 在论文中添加对照实验的定量结果

---

#### 问题 1.3：目标阈值的来源不明确

**现状**：
```
指标 | 目标范围 | 来源说明
-----|---------|----------
σ ≥ 4.0 | 无
α 1.5-2.5 | 无
C ≥ 0.30 | 无
L 2.0-3.5 | 无
```

**漏洞**：
- 这些目标数值是：
  - 从文献中提取？（引用号是？）
  - 从 C.elegans 实验数据？（WormAtlas 的哪个版本？）
  - 人为设定的"看起来合理的范围"？

**为什么严重**：
- 无法判断"达标"是否有物理意义
- 如果目标过宽松，虚假通过
- 如果目标过严格，虚假失败

**改进建议**：
- [ ] 为每个指标的目标添加引用
  ```
  σ ≥ 4.0: 根据 WormAtlas 302 神经元的度分布 [DOI:...]
  α 1.5-2.5: 文献中生物网络的典型范围 [小世界性质, Watts & Strogatz 1998]
  ```
- [ ] 检验目标是否与实际 C.elegans 重构数据匹配
- [ ] 灵敏度分析：目标偏离 ±10% 时，通过率如何变化？

### 类别 2：参数与物理约束（🟠 中等）

#### 问题 2.1：学习参数缺乏生物学基础

**现状**：
```python
# sdi_network_v30.py 中的典型参数
STDP_STRENGTH = 0.01
PLASTICITY_RATE = 0.05
DECAY_RATE = 0.1
THRESHOLD = 0.8
```

**漏洞**：
- 无法追溯这些参数的来源
- 是否与 C.elegans 的实际突触参数一致？
  - STDP 窗函数的时间常数：真实值是多少？
  - 突触衰减速率：是否与 τ_s ≈ 100-1000 ms 一致？
  - 放电阈值：是否与膜电位的生理范围一致？

**为什么中等**：
- 参数与结论相关，但"一般合理范围内"的参数不会完全破坏结果
- 但会影响量化的精度和可信度

**改进建议**：
- [ ] 为每个参数添加敏感性分析
  ```
  参数 | 默认值 | 扫描范围 | PASS/FAIL 转折点
  ----|--------|---------|---------------
  STDP_STRENGTH | 0.01 | 0.001-0.1 | ?
  DECAY_RATE | 0.1 | 0.01-0.5 | ?
  ```
- [ ] 从文献检索 C.elegans 的标准参数值
- [ ] 生成"参数稳健性曲线"显示结论对参数变化的敏感度

#### 问题 2.2：缺失物理约束验证

**现状**：
- 仿真中的突触权重、膜电位、放电速率等没有"上下界"检查
- 是否出现过"物理上不可能"的状态？

**具体漏洞**：
```
物理约束 | 是否检查 | 缺失风险
---------|---------|----------
膜电位范围: -70 to +30 mV | ❌ | 数值溢出
突触权重: [0, W_max] | ❌ | 无穷增长?
放电频率: [0, f_max] | ❌ | 超过生物范围?
传播延迟: 1-100 ms | ❌ | 不合理值?
```

**为什么中等**：
- 仿真通常"能跑完"意味着没有明显崩溃
- 但可能存在隐藏的数值不稳定性

**改进建议**：
- [ ] 在仿真循环中添加约束检查
  ```python
  assert -0.07 <= V_membrane <= 0.03, f"电压超出范围: {V_membrane}"
  assert 0 <= synaptic_weight <= 1.0, f"权重超出范围: {synaptic_weight}"
  ```
- [ ] 记录所有"约束违反"事件，纳入诊断报告

---

#### 问题 2.3：跨物种参数转移的正当性未证明

**现状**：
- v30 用同一套参数仿真 C.elegans（302 神经元）和 Drosophila（~100K 神经元）
- 假设：参数的相对关系（如时间常数比）保持不变

**漏洞**：
```
物种 | 神经元数 | 突触数 | 密度 | 假设同一参数是否合理?
----|---------|--------|------|--------------------
C.elegans | 302 | 7K | 7/300 = 2.3% | ✓ 小脑
Hemibrain | 100K | 20M | 20M/100K² ≈ 2% | ✓ 相近
鼠脑 | 100M | 100B | 100B/100M² ≈ 0.01% | ❌ 完全不同!
```

**为什么中等**：
- 当前仅在两个物种间转移，参数接近（密度 2-2.3%）
- 但如果要扩展到更大的系统，这个假设会失效

**改进建议**：
- [ ] 量化参数转移的有效范围
  ```
  有效范围：神经元数 10² - 10⁵，密度 1% - 5%
  超出范围需重新校准
  ```
- [ ] 进行"参数缩放律"分析：
  如果密度变为 10%，什么参数应该调整？

---

#### 问题 2.4：初始条件的敏感性未测试

**现状**：
- v30 使用"固定的 WormAtlas 拓扑"作为初始条件
- 从未测试：不同的随机初始化是否给出相同结果

**漏洞**：
```
实验设计缺失：
1. 同一初始拓扑 + 10 种不同的随机放电初值 → 收敛结果是否相同?
2. 不同初始拓扑（C.elegans 的多个个体）→ 结果是否稳定?
3. 初始学习速率扰动 → 最终状态是否吸引子?
```

**为什么中等**：
- 影响对"稳健性"和"通用性"的判断

**改进建议**：
- [ ] 设计"初始条件敏感性"实验
  - 保持拓扑，改变初始放电 100 次 → 计算收敛结果的方差
  - 改变初始拓扑 10 次 → 计算最终指标的 95% CI
- [ ] 在论文补充中呈现结果稳定性

### 类别 3：泛化性与推广（🟠 中等）

#### 问题 3.1："临界态"结论缺乏跨尺度验证

**现状**：
- 论文声称 iNEST 实现"自组织临界态"
- 但仅在两个物种尺度验证（302 和 100K 神经元）

**漏洞**：
```
尺度 | 测试 | 临界态指标 (γ_t, α, ...) | 是否一致?
-----|------|----------------------|----------
单个神经元 (1) | ❌ | N/A | —
局部回路 (10-50) | ❌ | ? | —
功能柱 (100-1000) | ❌ | ? | —
脑区 (1K-100K) | ✅ (仅C.elegans + Hemibrain) | 有数据 | 部分
全脑 (100K+) | ❌ | ? | —
```

**为什么中等**：
- "临界态"的定义本身要求"所有尺度上的幂律特性一致"
- 但多尺度验证本身缺失

**改进建议**：
- [ ] 对 C.elegans 的 7 个脑区（AL、MB、CX 等）分别计算
  - γ_t（时间复杂度）
  - γ_s（空间复杂度）
  - α（雪崩指数）
- [ ] 检验这些指标是否在各脑区间保持一致
- [ ] 绘制"缩放关系曲线"：系统大小 vs 指标值

### 类别 5：结论与声明的过度性（🔴 严重 + 🟠 中等）

#### 问题 5.1："5/5 PASS ✅" 的表述过于绝对

**现状**：
```
V25 论文中的声明：
"v30 仿真器达成 5/5 生物指标全部达标，验证了 iNEST 
SDI 架构在多物种、多尺度上的自组织临界态特性。"
```

**漏洞**：
- "全部达标"暗示高置信度，但
  - 无统计检验
  - 无对照实验
  - 无灵敏度分析

**改进建议**：
- [ ] 改为更严谨的表述
  ```
  "v30 仿真器在目标指标范围内完成 5/5 测试。
  在没有对照组的情况下，保留定论。
  后续工作应添加对照实验和统计显著性检验。"
  ```

---

#### 问题 5.2："验证"二字的使用过度

**现状**：
- 论文多处说"验证了 iNEST 的自组织临界态假设"
- 但实际上只是"仿真结果与预期一致"

**漏洞**：
```
逻辑错误：
假设：iNEST 实现 SOC
实验：仿真得到临界态指标
结论：假设验证

问题：
1. 没有反面实验（如果改参数，是否仍满足）
2. 没有对照（其他架构能否给出相同结果）
3. 没有硅基验证（仅基于软件仿真）
```

**改进建议**：
- [ ] 改为"假设一致性评估"而非"验证"
  ```
  "仿真结果与 SOC 假设相符"
  vs
  "验证了 SOC 假设" ❌
  ```
