## 【第 1 部分】仿真实验方法论总体框架

### 1.1 实验目标与假设

**主要假设**：
```
H0: iNEST SDI 架构实现自组织临界态 (Self-Organized Criticality, SOC)
    特征：度分布幂律分布 (σ ≥ 4.0)
         小世界特性 (α ∈ 1.5-2.5)
         高聚类系数 (C ≥ 0.30)
         短平均路径长度 (L ∈ 2.0-3.5)
         电-化学突触比 (EL ∈ 15-28%)
```

**验证方式**：
- 对真实生物网络（C.elegans、Drosophila）进行拓扑重构
- 运行神经放电仿真
- 检验结果指标是否满足上述条件

## 【第 2 部分】数据基础的完整回溯

### 2.1 数据来源的不匹配

**声称**：
> "验证了 iNEST SDI 在 C.elegans 和 Drosophila 上的适用性"

**实际**：

| 层面 | 声称 | 实际 | 差异 |
|-----|------|------|------|
| 1. 数据类型 | 真实连接组 | **部分合成** | 🔴 严重不符 |
| 2. 拓扑完整性 | 完整的生物拓扑 | 随机生成的参数 | 🔴 关键缺失 |
| 3. 动态过程 | 真实的神经放电 | 基于参数的生成 | 🔴 未实现 |

### 2.2 具体的数据问题

**问题 2.2.1：C.elegans 数据**
```
声称：使用 WormAtlas 302 神经元数据
实际：代码中的 region_sizes 定义了 13,200 个神经元
      这是 Drosophila 脑的规模，不是 C.elegans!
```

**问题 2.2.2：突触数量的生成逻辑**
```python
n_synapses = int(size * np.random.uniform(40, 80))
```

生物学对标：
- C.elegans: 302 神经元 × 平均 23 个连接/神经元 ≈ 7,000 突触
  - 比例: 7000 / 302 ≈ 23 (个/神经元)
  
- 代码生成: size × [40-80] = 13200 × [40-80]
  - 范围: 528,000 - 1,056,000 突触
  - 比例: [40-80] 个/神经元

**结论**：代码生成的突触数量**远远超过**生物数据10-20倍！

**问题 2.2.3：聚类系数的生成**
```python
clustering = float(np.random.beta(2, 5))
# 生成范围：[0, 1]，但生物网络通常 0.1-0.5
```

真实数据对标：
- C.elegans: C ≈ 0.24-0.32（文献值）
- 代码生成: C ∈ [0, 1]，无生物学约束

## 【第 3 部分】核心计算公式的詳細回顾

### 3.1 小世界指数计算

**公式**（从代码提取）：
```python
small_world = float(np.random.lognormal(1.5, 0.3))
# 或从 v30_results.json: "small_world": 7.197717
```

**定义检查**：
- 标准小世界指数: σ = (C/C_random) / (L/L_random)
- 通常 σ > 1 表示小世界
- 代码中：σ = 7.197 意味着什么？

**问题**：
- ❌ 代码未显示 C 和 L 的计算
- ❌ 未显示随机网络的生成
- ❌ 直接用对数正态分布随机生成，非计算所得

### 3.2 度分布与幂律指数

**目标指标**：σ ≥ 4.0（所谓的"平均度"）

**问题**：
```
术语混淆:
"σ" 在网络分析中通常表示：
  1. 平均度 (average degree)
  2. 小世界指数 (small-world sigma)
  
代码中使用哪一个？ 🤔 完全不清楚
```

**检验**：
```python
# 从 v30_results.json: "σ（平均度） | 4.238 | ≥4.0 | ✅"
# 这是平均度还是小世界指数？

# 如果是平均度：
# 平均度 = 2 × (突触数) / (神经元数)
# 例: C.elegans: 2 × 7000 / 302 ≈ 46.4

# 但 σ = 4.238 < 46.4，不匹配！
```

### 3.3 放电动力学（如果有的话）

**问题**：
- ❌ 代码中完全看不到神经放电仿真
- ❌ 没有 Hodgkin-Huxley、Integrate-and-Fire 等模型
- ❌ "3000步 MNIST" 是怎么做的？**代码中未实现！**

**声称 vs 实际**：
```
论文声称：
"运行 3000 步 MNIST 分类任务的神经放电仿真"

代码实际：
- 只有拓扑指标计算
- 无时间动力学仿真
- 无学习（STDP）过程
- 无行为输出
```

### 4.2 逻辑断裂点

**断裂 1: 从拓扑到功能**
```
给定：σ=4.238, 这代表什么？

若 σ 是平均度：
  - 计算: 度数 = 2×边数/节点数
  - 对 C.elegans: 度 ≈ 46
  - 但声称 σ = 4.238 → 不匹配！

若 σ 是小世界指数：
  - σ = 7.2 (从 small_world 字段)
  - 表示"相比随机网络更小世界"
  - 但未计算随机网络对照 → 无法验证
```

**断裂 2: 从静态拓扑到动态过程**
```
现状：只有拓扑指标（无时间依赖）
声称：实现了 SOC（需要时间动力学）
问题：拓扑的小世界性 ≠ 时间上的临界态
      缺少：放电雪崩分布、脉冲传播、学习可塑性
```

**断裂 3: 从仿真到硬件**
```
现状：Python 脚本生成的随机指标
声称：SDI 硬件架构优于传统芯片
问题：仿真不等于硬件实现
      缺少：功耗、延迟、面积的实际测量
```

## 【第 10 部分】正确的方法论应该是这样的

### 10.1 正确流程

```
第一阶段：数据准备
  ├─ 加载真实连接组 (WormAtlas、Hemibrain)
  ├─ 验证数据完整性 (节点数、边数、度分布)
  ├─ 绘制拓扑特性 (可视化验证)
  └─ 生成对照网络 (3-5 种 Null model)

第二阶段：静态拓扑分析
  ├─ 计算拓扑指标 (使用标准公式)
  ├─ 与对照网络对比 (KL 散度、Wasserstein)
  ├─ 统计检验 (t-test、KS test)
  └─ 多尺度分析 (子图级别的重复)

第三阶段：动态仿真
  ├─ 实现神经放电模型 (HH 或 LIF)
  ├─ 添加噪声和随机性
  ├─ 运行任务 (MNIST、运动控制)
  └─ 记录放电时间序列

第四阶段：功能指标
  ├─ 计算放电雪崩 (P(s) ∝ s^(-α))
  ├─ 功率谱分析 (P(f) ∝ f^(-γ))
  ├─ 信息论指标 (熵、互信息)
  └─ 任务性能 (精度、延迟)

第五阶段：对比与结论
  ├─ 与对照组对比
  ├─ 与文献对比
  ├─ 结论：支持/反驳 SOC 假设
  └─ 适用范围声明
```
