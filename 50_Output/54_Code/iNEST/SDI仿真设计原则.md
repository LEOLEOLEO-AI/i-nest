## 核心原则：真实公开数据集严格匹配

**所有仿真计算必须基于真实公开连接组数据集，与特定物种和特定数据集严格匹配，才能向后执行代码。**

### 禁止行为（红线，任何情况下不得违反）

1. ❌ **禁止使用合成网络替代真实数据**
   - 不得使用 `nx.watts_strogatz_graph()`、`nx.barabasi_albert_graph()` 等随机图模型代替真实连接组
   - 不得用 `N=2000` 模拟 C. elegans（真实 N=302），即使 k/L/C 参数来自文献

2. ❌ **禁止在代码中硬编码文献参数替代实测值**
   - 不得将 `L=2.46`、`C=0.308` 等文献值直接写入配置并跳过实际计算
   - 参数必须从真实数据集中实际测量得到

3. ❌ **禁止以"大图采样"方式随意截断数据**
   - 如需采样，必须说明采样方法、比例、随机种子，并在结果中注明"采样结果"

4. ❌ **禁止混用不同物种的参数**
   - C. elegans k≈14 的数据不得用于 Fly Hemibrain 的合理性判断

### 强制要求

1. ✅ **每个物种必须对应唯一、可溯源的公开数据集**，在代码中注明：
   - 数据集名称
   - 原始论文 DOI
   - 下载来源 URL
   - 数据版本

2. ✅ **所有网络参数（N, k, L, C, S）必须从真实数据实测**，不得预设

3. ✅ **数据读取必须包含必要的预处理**，并在注释中说明：
   - 多重边处理方式（如 Hemibrain 的跨ROI有向边合并）
   - 阈值化方式（如 Budapest DTI 的 occurence 阈值）
   - 自环处理
   - LCC 提取

4. ✅ **如果真实数据集不可用**，该物种必须从仿真中**完全排除**，不得用合成数据占位

### 数据缺口处理规则

- 数据集缺失 → 物种从当轮仿真**排除**，结果中注明"数据待补"
- 不得用合成网络占位
- 不得将缺失物种的 RI 设为固定值（如 `RI=1.618`）

---

**此原则适用于所有后续 CST 仿真代码，包括区域分析、多物种对比、PPT图生成等。**
