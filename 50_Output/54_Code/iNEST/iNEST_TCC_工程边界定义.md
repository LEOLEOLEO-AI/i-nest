# iNEST与TCC的工程边界定义

## 一句话区分

| 子项目 | 核心命题 | 工程目标 |
|--------|---------|---------|
| **iNEST** | 连接复杂度**何时**、**为何**产生智能涌现 (1+1>N) | 涌现科学的工程验证 |
| **TCC** | **如何**工程化实现拓扑中心计算 (1+1>2) | 拓扑计算的硬件落地 |

## 工程任务归属

| 任务 | 归属 | 理由 |
|------|------|------|
| CST数值仿真 | **iNEST** | 理论验证, 非硬件工程 |
| 神经雪崩实验 | **iNEST** | 生物学对照, 非硬件工程 |
| 忆阻器阵列临界态测试 | **iNEST** | 物理涌现验证, 非产品化 |
| CST参数→SDI映射 | **iNEST→TCC** | 桥梁, 双方共同 |
| NPC原语RTL设计 | **TCC** | 硬件实现 |
| SDI交换芯片 | **TCC** | 硬件实现 |
| FPGA原型验证 | **TCC** | 硬件实现 |
| Chiplet流片 | **TCC** | 硬件实现 |
| TCCL编译器 | **TCC** | 软件实现 |
| AI+信号处理Demo | **TCC** | 产品化 |
| FFT-AllReduce同构验证 | **TCC** | 硬件工程(但受益于iNEST理论) |
| 涌现度量工具链 | **iNEST** | 科学工具, 非产品 |

## 共享资源

- 飞书NICE知识库 → 双方共用
- Get笔记知识库 → 双方共用
- FPGA开发平台 → 天大管理, 双方共用
- GPU计算集群 → iNEST优先, TCC可排队使用
