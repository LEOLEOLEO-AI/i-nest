# ADR: 以 TCC + SDI-CC 作为 2026-2027 近期工程主线

## 背景

当前工作区中的项目、论文与规划文档很多，但近期资源和时间窗口有限，不能同时推进所有方向。

现有材料显示，最适合在 2026-2027 年形成“可申报、可原型、可投稿、可讲故事”闭环的主线，是：

- `TCC`：网络中心计算，负责范式定义
- `SDI-CC`：软件定义互连与集合通信原语映射，负责工程抓手

这条主线已经同时具备：

- 项目载体：20_Projects/Projects/海河实验室重大专项/海河实验室_项目申报书_晶上网络中心计算_完整版
- 理论文稿：30_Outputs/论文/B组_SDI-CC互连体系/P-Theory_v2_MetaTopology_SDI_Bond_Draft
- 映射文稿：30_Outputs/论文/B组_SDI-CC互连体系/[V2] 论文框架_P-Mapping_5plus4完备物理拓扑映射
- 范式叙事：30_Outputs/论文/B组_SDI-CC互连体系/SDI-CC论文框架_拓扑即计算新范式

## 决策

在 2026-2027 近期工程阶段，统一采用以下主线：

- 项目主线：以 `海河实验室重大专项` 为近期工程抓手
- 理论主线：以 `TCC` 第三计算范式和 `SDI-Bond / Meta-Topology` 为理论支撑
- 原型主线：以 `3+1` 极简 MVP、多 FPGA 原型、热切换演示为验证路径
- 论文主线：优先推进 `B1 / B2 / B3`

实施时要求：

- 申报书、汇报、论文中对 `TCC / SDI / 5+4 原语 / SDIO-N` 的表述保持一致
- 所有近期原型工作优先服务于 `B1/B2/B3` 的投稿与项目答辩
- 卫星智能体等场景化方向，优先复用此主线能力，不单独另起底层架构

## 备选方案

1. 以 `CST` 纯理论论文作为唯一近期主线

缺点：

- 理论强，但近期工程抓手弱
- 不利于快速形成可演示原型和专项申报故事

2. 以 `卫星智能体` 场景作为唯一近期主线

缺点：

- 场景吸引人，但底层技术仍依赖 `TCC + SDI-CC`
- 容易先讲场景、后补理论，导致主线倒挂

3. 同时平推多条主线

缺点：

- 资源被摊薄
- 项目、论文、技术开发无法形成闭环

## 影响

正面影响：

- 统一近期叙事，减少概念分叉
- 项目、论文、技术开发三线能够形成强耦合推进
- 有利于形成近期可见成果：专项、原型、论文、演示系统

约束影响：

- 其他方向暂时不作为近期主线，只保留为储备池
- 场景化方向必须以复用主线底座为原则

落地影响：

- 20_Projects/00_项目推进面板 的 `P0` 工作优先围绕海河主线展开
- 30_Outputs/论文/00_论文优先级推进面板 中 `B1/B2/B3` 进入近期重点
- 90_System/00_技术开发推进面板 中原型、SDK、导入与面板工具应服务这条主线
