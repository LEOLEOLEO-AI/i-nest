# SDI集合通信硬件化方案 v1.0

## 总体架构

### 三层结构
```
Layer 3: 计算芯粒（Compute Die）
  NPU/GPU/AI Core + 本地SRAM
  每芯粒配1个HCI（硬通信接口）
  职责：发原始数据、收最终结果，不参与通信逻辑

Layer 2: 互连芯粒（Interconnect Die）← 核心
  硬路由器阵列 + HCU（硬集合通信单元）
  内置：
    - 硬Allreduce引擎（FP16/BF16/INT8，Sum/Min/Max）
    - 硬Alltoall引擎（分发/收集/重排）
    - 硬归约树（多输入→单输出，全流水，周期级延迟）
    - 硬广播树（单输入→多输出，硬复制）
    - 硬全交换Crossbar（N×N，无阻塞）

Layer 1: 晶圆级背板（Wafer-scale Backplane）
  硅中介层/TSV/微凸点/光互连
  单链路：2-8 TB/s
  SDI直接配置链路通断/带宽/拓扑
```

### SDI控制平面（极简三件事）
```
1. 拓扑编排：Ring/Tree/Mesh/Hybrid动态切换
2. 映射下发：rank → 物理路由器 → 物理链路
3. 资源分配：带宽/VC/优先级/硬单元分片

SDI不做：流控、路由计算、负载均衡、重传、排序（全硬做）
```

## 性能指标（硬件实现 vs 软件实现）

| 指标 | 软件（TCCL+RoCE） | 硬件（本方案） | 提升倍数 |
|------|-----------------|--------------|---------|
| Allreduce延迟（8芯粒） | 50-200μs | 0.2-0.8μs | 250-1000× |
| Allreduce带宽利用率 | 40-60% | 100% | 1.6-2.5× |
| Alltoall延迟（16芯粒） | 100-500μs | 0.5-2μs | 200-500× |
| 算力占用（通信） | 30-50% GPU时间 | 0% | 全部用于计算 |
| 延迟方差 | 30-100% | <5% | 极稳定 |
| 功耗降低 | 基准 | 30-50%↓ | — |

## 流片规格（第一版目标）

```
工艺：28nm（中芯国际/华虹，学术MPW通道）
互连芯粒面积：4mm × 4mm
硬归约树：8输入，FP16/BF16，全流水
硬Crossbar：8×8，无阻塞
SDI接口：3张配置表（拓扑/映射/资源），AXI4-Lite
功耗目标：<5W（互连芯粒单片）
```

## FPGA原型方案（第一步验证）

```
平台：Xilinx Alveo U280 或 Intel Stratix 10
实现规模：4芯粒硬Ring Allreduce + 4×4硬Mesh Alltoall
验证目标：
  - Allreduce延迟 < 2μs（FPGA上，流片后<0.5μs）
  - 带宽利用率 > 85%
  - SDI配置切换延迟 < 10μs
时间计划：M3-M4（3-4个月后）
```
