# SDSoW 硬件映射方案

## Software-Defined System on Wafer — SDI 三规则硬件原语映射

**文档版本**：v1.0  
**日期**：2026-05-08  
**机构**：iNEST — Institute for Neuromorphic Engineering & Self-organizing Technologies  
**基础实验**：实验一（C.elegans N=279，5/5生物指标达标）、实验二（嗅觉系统 N=1351，KC稀疏编码2.55%）  

---

> **核心命题**：大道至简（Complexity comes from simplicity）  
> 把实验验证的三条极简SDI规则，固化为SDSoW硬件原语，让硅基网络自发涌现从线虫到超人类的脑智能。

## 第一部分：SDI 规则的硬件原语映射

### 1.1 规则一：STDP（Spike-Timing Dependent Plasticity，脉冲时序依赖可塑性）

#### 仿真参数（实验验证基础）

```
ETA_LTP  = 0.012 ~ 0.015   # LTP 学习率
TAU_STDP = 20               # STDP 时间常数（步）
THETA_LTP = 65 ~ 80         # LTP 固化阈值
THETA_LTD = 15              # LTD 消除阈值
ETA_LTD  = 0.008            # LTD 学习率
```

#### 数学模型

$$
\Delta w = \begin{cases}
+\eta_{LTP} \cdot e^{-|\Delta t| / \tau_{STDP}} & \text{if } t_{pre} < t_{post} \text{ (LTP)} \\
-\eta_{LTD} \cdot e^{-|\Delta t| / \tau_{STDP}} & \text{if } t_{pre} > t_{post} \text{ (LTD)}
\end{cases}$$

#### 硬件原语设计

**① 时间戳寄存器（Spike Timestamp Register）**

```
┌─────────────────────────────────────────┐
│  Pre-synaptic Timestamp Register        │
│  Width: 16-bit (max 65535 time steps)   │
│  Reset: on each pre-spike               │
│                                         │
│  Post-synaptic Timestamp Register       │
│  Width: 16-bit                          │
│  Reset: on each post-spike              │
│                                         │
│  Δt Compute: Δt = T_post - T_pre       │
│  Range: [-32768, +32767]               │
└─────────────────────────────────────────┘
```

**② 指数衰减电路（Exponential Decay Circuit）**

```
┌─────────────────────────────────────────────────────┐
│  STDP Exponential Kernel (exp(-|Δt|/τ))             │
│                                                     │
│  实现方案A（查找表LUT）：                             │
│  - 预计算 exp(-k/20) for k = 0..127                 │
│  - 128-entry × 8-bit LUT                           │
│  - 面积: ~4 logic cells                             │
│                                                     │
│  实现方案B（电容RC近似）：                            │
│  - 充放电时间常数 RC = τ_STDP / f_clock             │
│  - 适合模拟混合信号工艺                               │
│  - 精度: ~5%（足够神经形态计算）                     │
└─────────────────────────────────────────────────────┘
```

**③ 权重累积计数器（Weight Accumulation Counter）**

```
┌─────────────────────────────────────────────────────┐
│  Per-synapse STDP Accumulator                       │
│                                                     │
│  LTP_counter: 8-bit saturating counter              │
│    +1 per LTP event (|Δt| < τ, pre before post)    │
│    Reset: on weight potentiation                    │
│    Threshold: THETA_LTP = 65 ~ 80                   │
│                                                     │
│  LTD_counter: 8-bit saturating counter              │
│    +1 per LTD event (|Δt| < τ, post before pre)    │
│    Reset: on weight depression                      │
│    Threshold: THETA_LTD = 15                        │
│                                                     │
│  Weight update logic:                               │
│    if LTP_cnt >= THETA_LTP: w += ETA_LTP, reset     │
│    if LTD_cnt >= THETA_LTD: w -= ETA_LTD, reset     │
└─────────────────────────────────────────────────────┘
```

**④ SDI Bond 结构（每条物理互连的最小单元）**

```
┌─────────────────────────────────────────────────────┐
│  SDI Bond Hardware Structure (per interconnect)     │
│                                                     │
│  ┌──────────┬──────────┬────────────┬───────────┐  │
│  │ 4-bit    │ 8-bit    │ 8-bit      │ 16-bit    │  │
│  │ Weight   │ LTP cnt  │ LTD cnt    │ Timestamp │  │
│  │ Register │          │            │ Register  │  │
│  └──────────┴──────────┴────────────┴───────────┘  │
│  Total: 36-bit state per bond                       │
│  Logic cells: ~50 cells/bond (28nm工艺)              │
│  面积: ~0.25μm² × 50 = 12.5μm² per bond            │
└─────────────────────────────────────────────────────┘
```

### 1.2 规则二：WS 随机重连（Watts-Strogatz Rewiring）

#### 仿真参数（实验验证基础）

```
P_REWIRE   = 0.15    # 重连概率（每次检查）
REWIRE_INT = 50      # 重连间隔（仿真步数）
```

#### 数学模型

每 `REWIRE_INT` 步，对所有空闲（或弱）互连 bond，以概率 `P_REWIRE` 将其目标端点切换到随机选取的非邻居节点。

#### 硬件原语设计

**① 可重构交叉开关矩阵（Reconfigurable Crossbar）**

```
┌──────────────────────────────────────────────────────────┐
│  Reconfigurable Crossbar (N×K matrix)                    │
│  N = 神经元数量, K = 最大fan-out per neuron (≈ 16)       │
│                                                          │
│  物理结构：                                               │
│  ┌─────────┬─────────────────────────────────────────┐  │
│  │ Source  │ Target Routing Table                    │  │
│  │ Neuron  │ [target_0][target_1]...[target_K]       │  │
│  │ ID      │ Each entry: log2(N)-bit target address  │  │
│  └─────────┴─────────────────────────────────────────┘  │
│                                                          │
│  重配置端口：                                             │
│  - cfg_src: 源神经元地址                                  │
│  - cfg_bond_idx: 被重连的bond索引                         │
│  - cfg_new_tgt: 新目标神经元地址                           │
│  - cfg_valid: 触发信号                                    │
│                                                          │
│  延迟要求：< 1个时间步（~1μs @ 1MHz仿真时钟）              │
└──────────────────────────────────────────────────────────┘
```

**② 线性反馈移位寄存器（LFSR 随机数生成器）**

```
┌──────────────────────────────────────────────────────────┐
│  LFSR Random Number Generator                            │
│                                                          │
│  规格：                                                   │
│  - 长度: 32-bit（周期 2^32 - 1 ≈ 43亿）                  │
│  - 多项式: x^32 + x^22 + x^2 + x^1 + 1 (Galois型)       │
│  - 输出: 每时钟周期产生1个随机bit                          │
│                                                          │
│  HDL伪代码（Verilog风格）：                               │
│                                                          │
│  reg [31:0] lfsr;                                        │
│  wire feedback = lfsr[31]^lfsr[21]^lfsr[1]^lfsr[0];    │
│  always @(posedge clk) begin                             │
│    lfsr <= {lfsr[30:0], feedback};                       │
│    if (rewire_trigger)                                   │
│      rand_target <= lfsr[log2(N)-1:0] % N;              │
│  end                                                     │
│                                                          │
│  概率映射：P_REWIRE=0.15 → 阈值 = 0.15 × 2^32 ≈ 644M    │
│  比较：if (lfsr < 644M) → 触发重连                        │
└──────────────────────────────────────────────────────────┘
```

**③ 重连触发器（Rewiring Controller FSM）**

```
┌──────────────────────────────────────────────────────────┐
│  Rewiring Controller (Finite State Machine)              │
│                                                          │
│  状态机：                                                 │
│  IDLE → SCAN_BONDS → CHECK_PROB → SELECT_TARGET →        │
│          EXECUTE_REWIRE → UPDATE_TABLE → IDLE            │
│                                                          │
│  触发条件：                                               │
│  - 全局时钟计数器 % REWIRE_INT == 0                       │
│                                                          │
│  Idle Bond 判定：                                         │
│  - 权重 w < w_min_threshold (默认 0.05)                  │
│  - 或：近期（最近REWIRE_INT步）无LTP事件                   │
│                                                          │
│  目标选择约束：                                           │
│  - 排除已连接的邻居（collision check via bit-vector）     │
│  - 排除自连接                                             │
│  - 最多重试3次，超时保持原连接                             │
│                                                          │
│  关键时序：整个重连过程 < 50 clock cycles                  │
│  @ 1MHz时钟 → < 50μs（满足 <1个时间步 约束）              │
└──────────────────────────────────────────────────────────┘
```

**④ SDSoW "液态重构"核心机制**

```
┌──────────────────────────────────────────────────────────┐
│  SDSoW Liquid Reconfiguration Core                       │
│                                                          │
│  这是SDSoW区别于所有现有神经形态芯片的核心差异：            │
│  物理互连（bond）可以在运行时动态重定向                     │
│                                                          │
│  硬件实现路径：                                           │
│  1. 粗粒度：可编程开关矩阵（PSM, Programmable Switch      │
│             Matrix）—— 类FPGA的可重配路由资源             │
│  2. 细粒度：每个bond有独立的目标地址寄存器                  │
│             可在1个时间步内原子更新                        │
│                                                          │
│  "液态"的数学含义：                                       │
│  拓扑矩阵 A(t) 是时间的函数，而非静态固定值               │
│  每次WS重连 = 修改 A(t) 中的一个非零元素位置              │
└──────────────────────────────────────────────────────────┘
```

### 1.3 规则三：突触缩放（Homeostatic Plasticity / Synaptic Scaling）

#### 仿真参数（实验验证基础）

```
SCALING_INT = 100    # 缩放间隔（仿真步数）
激活率 > 25%: 下调权重（抑制过活跃神经元）
激活率 < 5%:  上调权重（激活沉默神经元）
```

#### 数学模型

$$
w_{ij}(t+1) = w_{ij}(t) \times \begin{cases}
\alpha_{down} < 1 & \text{if } r_i > 0.25 \\
\alpha_{up} > 1   & \text{if } r_i < 0.05 \\
1                 & \text{otherwise}
\end{cases}$$

其中 $r_i$ 是神经元 $i$ 在过去 `SCALING_INT` 步内的平均激活率。

#### 硬件原语设计

**① 激活率计数器（Activity Rate Counter）**

```
┌──────────────────────────────────────────────────────────┐
│  Per-neuron Activity Rate Counter                        │
│                                                          │
│  结构：滑动窗口计数器                                      │
│  - 窗口宽度：SCALING_INT = 100 步                         │
│  - 计数宽度：8-bit（0~255，但实际最大100）                  │
│  - 实现：环形缓冲器 OR 指数移动平均（EMA）                  │
│                                                          │
│  EMA实现（硬件友好）：                                     │
│  rate_reg <= (1 - α) * rate_reg + α * spike_in           │
│  α ≈ 1/SCALING_INT = 0.01                                │
│  rate_reg: 16-bit fixed-point (8.8格式)                  │
│                                                          │
│  面积: ~20 logic cells per neuron                        │
│  For N=2K neurons: 2K × 20 = 40K logic cells            │
└──────────────────────────────────────────────────────────┘
```

**② 比较器电路（Threshold Comparator）**

```
┌──────────────────────────────────────────────────────────┐
│  Activity Threshold Comparator                           │
│                                                          │
│  输入：rate_reg (8.8 fixed-point)                        │
│  阈值寄存器（可编程）：                                    │
│    HI_THRESH = 0.25 × 256 = 64  (8.8格式整数部分)        │
│    LO_THRESH = 0.05 × 256 = 13                           │
│                                                          │
│  输出三态标志：                                           │
│    scaling_flag[1:0]:                                    │
│      2'b10 → SCALE_DOWN (rate > 25%)                    │
│      2'b01 → SCALE_UP   (rate < 5%)                     │
│      2'b00 → NO_SCALE   (稳态区间)                       │
│                                                          │
│  每 SCALING_INT 步触发一次全网络扫描                       │
│  可用流水线方式：每步处理 N/SCALING_INT 个神经元           │
└──────────────────────────────────────────────────────────┘
```

**③ 全局权重缩放单元（Global Weight Scaling Unit）**

```
┌──────────────────────────────────────────────────────────┐
│  Global Weight Scaling Engine                            │
│                                                          │
│  功能：批量调整单个神经元的所有入突触权重                   │
│                                                          │
│  硬件实现：乘法器阵列 + DMA访问权重存储                    │
│    - 缩放因子 α: 可编程，8-bit精度                        │
│    - SCALE_DOWN: α_down = 0.95 (可调)                    │
│    - SCALE_UP:   α_up   = 1.05 (可调)                    │
│                                                          │
│  批处理流程：                                             │
│    每触发一次：读入→乘以α→写回→下一条突触                  │
│    吞吐量: 1 synapse/cycle × 平均K=16条突触/神经元         │
│    总延迟: N × K / f_clock = 2K×16 / 1MHz = 32ms         │
│    （满足 <100μs 需流水线加速或更高时钟）                   │
│                                                          │
│  SDI 分层缩放架构：                                       │
│    局部缩放（Local）: 调整单个神经元所有入突触              │
│    全局缩放（Global）: 跨层次的整体权重归一化               │
│    触发优先级: Local 先于 Global（防止震荡）               │
└──────────────────────────────────────────────────────────┘
```

### 各层功能详述

#### 层1：硅基底层（Silicon Substrate Layer）

| 子模块 | 功能 | 关键参数 |
|--------|------|----------|
| 神经元阵列 | LIF（Leaky Integrate-and-Fire）脉冲神经元计算 | ≥2K tiles，每tile 1个神经元 |
| 突触存储 | 稀疏权重矩阵存储（仅存非零突触） | SRAM，N×K×4bit |
| 绝对不应期 | 脉冲后锁定窗口（T_ABS=3步） | 3-bit计数器/neuron |
| 相对不应期 | 脉冲后抑制窗口（T_REL=8步） | 衰减因子寄存器 |
| 时钟/复位 | 全局同步时钟 + 分布式异步事件驱动 | 1MHz基础时钟 |

**接口**：脉冲输出（spike_out）→ 上层事件总线；权重读写接口 → 物理层权重寄存器阵列

---

#### 层2：SDSoW 物理层（Flexible Bond Layer）

**核心创新**：这一层是SDSoW区别于所有现有神经形态芯片的根本所在——互连是"液态"的，可以在运行时任意重配置。

| 子模块 | 功能 | 关键参数 |
|--------|------|----------|
| 可重构互连矩阵 | 实现WS随机重连的物理基础 | N×K crossbar，重配置延迟<1μs |
| 权重寄存器阵列 | 每条bond的4-bit权重 + STDP计数器 | 36-bit/bond，~50 logic cells/bond |
| 事件总线 | 脉冲时间戳广播，触发STDP计算 | 低延迟环形总线，<1 clock latency |
| 激活率监测器 | 滑动窗口激活统计，触发突触缩放 | 16-bit EMA/neuron |

**接口**：
- 下行：读取神经元脉冲输出（from 底层）
- 上行：向规则层提供网络状态摘要（激活率、拓扑统计）
- 侧向：bond重配置控制信号

---

#### 层3：SDI 规则层（Rule Engine Layer）

三条规则的协同执行逻辑，本质是三个独立的硬件状态机，共享访问物理层资源：

```
┌─────────────────────────────────────────────────────────────┐
│  规则协同仲裁器（Rule Arbitration Unit）                      │
│                                                             │
│  时间片调度：                                               │
│  步 t mod 1   → STDP 检查（每步都执行）                     │
│  步 t mod 50  → WS 重连检查（REWIRE_INT=50）                │
│  步 t mod 100 → 突触缩放触发（SCALING_INT=100）             │
│                                                             │
│  冲突解决：WS重连与STDP同时触发时，STDP优先                  │
│  （保护刚刚被强化的突触不被立刻重连消除）                     │
└─────────────────────────────────────────────────────────────┘
```

## 第六部分：核心创新点总结（专利/论文要点）

### 6.1 创新点一：SOC-Native 架构（自组织临界原生支持）

**技术描述**：  
现有神经形态芯片（Loihi2, TrueNorth, SpiNNaker）均无法在不依赖外部软件的情况下将网络自主维持在临界态（SOC）。SDSoW通过将三规则协同固化为硬件FSM，使芯片**天然收敛**到σ≥4的小世界临界态。

**实验依据**：  
实验一中，从任意随机初始网络出发，三规则协同在<200步内将C.elegans网络驱动至σ=7.63（目标≥4.0），无需任何外部调参。

**专利要点**：  
权利要求：一种包含STDP固化/消除单元、WS随机重连控制器和突触缩放引擎的神经形态处理器，其特征在于三个硬件模块的协同调度使网络自主维持小世界系数σ≥4的临界态。

---

### 6.2 创新点二：液态拓扑互连（Liquid Topology）

**技术描述**：  
SDSoW的物理互连（bond）携带独立的目标地址寄存器，可在运行时以单步精度（~1μs）重定向到任意新目标。这是第一个在硬件层面原生支持WS随机重连算法的神经形态系统。

**实现关键**：  
N×K可重构交叉开关矩阵 + LFSR随机数生成器 + 重连控制FSM，三者协同实现"液态"的物理拓扑。

**专利要点**：  
权利要求：一种神经形态处理器中的可重构互连结构，其特征在于每条突触互连包含运行时可写的目标地址寄存器和权重累积计数器，并配备随机数生成器驱动的自动重连控制器。

---

### 6.3 创新点三：三规则统一硬件实现

**技术描述**：  
现有芯片通常只支持一种可塑性规则（Loihi2仅STDP，TrueNorth无可塑性）。SDSoW在同一芯片上原生实现三条规则的协同硬件执行，并通过时间片仲裁机制避免冲突。

**硬件证明**：  
面积开销：STDP模块~1.6M cells + WS重连~10K cells + 突触缩放~20K cells，三者之和在28nm工艺下约5mm²，对比SpiNNaker2（10.6mm²）面积更小。

## 附录A：关键 HDL 伪代码片段

### A.1 STDP Bond 模块（Verilog伪代码）

```verilog
module stdp_bond #(
    parameter TAU_STDP  = 20,   // 时间常数（步数）
    parameter ETA_LTP   = 12,   // LTP学习率 ×1000 (0.012)
    parameter THETA_LTP = 65,   // LTP固化阈值
    parameter THETA_LTD = 15    // LTD消除阈值
)(
    input  clk, rst_n,
    input  pre_spike, post_spike,
    output reg [3:0] weight,    // 4-bit权重
    output bond_prune           // 消除信号
);
    // 时间戳寄存器
    reg [15:0] pre_ts, post_ts;
    reg [7:0]  ltp_cnt, ltd_cnt;
    
    // LUT: exp(-k/20) × 256, for k=0..127
    reg [7:0] exp_lut [0:127];
    
    // 脉冲时间差计算
    wire [15:0] delta_t = post_ts - pre_ts;
    wire [6:0]  abs_dt  = (delta_t[15]) ? (~delta_t[6:0]+1) : delta_t[6:0];
    wire        is_ltp  = !delta_t[15];   // pre在post之前
    
    always @(posedge clk) begin
        if (pre_spike)  pre_ts  <= global_timer;
        if (post_spike) post_ts <= global_timer;
        
        // LTP事件累积
        if (pre_spike | post_spike) begin
            if (is_ltp && abs_dt < 64)
                ltp_cnt <= ltp_cnt + exp_lut[abs_dt];
            else if (!is_ltp && abs_dt < 64)
                ltd_cnt <= ltd_cnt + exp_lut[abs_dt];
        end
        
        // 阈值触发权重更新
        if (ltp_cnt >= THETA_LTP) begin
            weight  <= weight + 1;  // 饱和加
            ltp_cnt <= 0;
        end
        if (ltd_cnt >= THETA_LTD) begin
            weight  <= weight - 1;  // 饱和减
            ltd_cnt <= 0;
        end
    end
    
    assign bond_prune = (weight == 4'b0000) && (ltd_cnt >= THETA_LTD);
endmodule
```

### A.2 WS 重连控制器（Verilog伪代码）

```verilog
module ws_rewire_ctrl #(
    parameter N            = 2048,  // 神经元数
    parameter REWIRE_INT   = 50,    // 重连间隔
    parameter P_REWIRE_THR = 32'd644245094  // 0.15 × 2^32
)(
    input  clk, rst_n,
    input  bond_prune_i,     // 来自STDP模块的消除信号
    input  [10:0] src_id,    // 源神经元ID
    output reg [10:0] new_tgt, // 新目标神经元ID
    output reg cfg_valid      // 重配置触发
);
    reg [31:0] lfsr;
    reg [15:0] step_cnt;
    
    // Galois LFSR
    wire feedback = lfsr[31]^lfsr[21]^lfsr[1]^lfsr[0];
    
    always @(posedge clk) begin
        lfsr     <= {lfsr[30:0], feedback};
        step_cnt <= step_cnt + 1;
        cfg_valid <= 0;
        
        if (bond_prune_i && (lfsr < P_REWIRE_THR)) begin
            new_tgt   <= lfsr[10:0] % N;  // 随机目标
            cfg_valid <= 1;
        end
    end
endmodule
```

### A.3 突触缩放引擎（伪代码）

```verilog
module synaptic_scaling #(
    parameter N           = 2048,
    parameter SCALING_INT = 100,
    parameter HI_THRESH   = 8'd64,  // 25% × 256
    parameter LO_THRESH   = 8'd13   // 5%  × 256
)(
    input  clk, rst_n,
    input  spike_in,
    input  [10:0] neuron_id,
    output reg scale_down, scale_up,
    output reg [10:0] scale_target_id
);
    // EMA 激活率寄存器（每神经元16-bit，8.8格式）
    reg [15:0] rate_ema [0:N-1];
    reg [15:0] step_cnt;
    
    // α = 1/SCALING_INT = 0.01（8.8格式: 256/100 ≈ 3）
    localparam ALPHA = 8'd3;
    
    always @(posedge clk) begin
        // 更新激活率 EMA
        if (spike_in)
            rate_ema[neuron_id] <= rate_ema[neuron_id] - (rate_ema[neuron_id]>>8)*ALPHA
                                  + ALPHA;
        else
            rate_ema[neuron_id] <= rate_ema[neuron_id] - (rate_ema[neuron_id]>>8)*ALPHA;
        
        // 每SCALING_INT步扫描并触发缩放
        if (step_cnt % SCALING_INT == 0) begin
            scale_target_id <= neuron_id;
            if (rate_ema[neuron_id][15:8] > HI_THRESH) scale_down <= 1;
            if (rate_ema[neuron_id][15:8] < LO_THRESH) scale_up   <= 1;
        end
    end
endmodule
```

## 附录B：SOC 临界态硬件检测

### B.1 σ 小世界系数实时计算模块

为支持在线SOC监测，需要一个硬件加速的σ估算模块：

```
σ = (C/p) / (L/L_rand)

其中：
  C     = 聚集系数（三角形计数，BFS加速）
  p     = 网络密度（已知，由连接数/N(N-1)计算）
  L     = 平均路径长度（采样BFS，随机抽40节点）
  L_rand = ln(N)/ln(K)（随机网络理论值，可直接计算）
```

**硬件实现路径**：
- 聚集系数：矩阵乘法加速（N×N稀疏矩阵，每步采样计算，不需全网络精确值）
- 路径长度：BFS流水线，每4个时钟步处理一个BFS层
- σ：定点除法器，8-bit精度已足够判断是否≥4.0

## 第七部分：实验五验证的SOC动力学硬件原语（四规则体系扩展）

> **版本说明**：实验五（v12最终版）在三规则基础上引入**软饱和STDP + 稳态权重归一化 + BTW极慢驱动 + 双重临界点校准**，形成四规则体系。本章节将四规则体系对应的硬件原语系统化地映射到SDSoW芯片设计约束。

### 7.1 实验五核心参数与硬件实现支撑

#### 7.1.1 软饱和STDP（BCM 1982 + Bhatt 2009 Neuron）

**数学定义**

$$dw_{\text{eff}} = dw \times \left(1 - \frac{w}{W_{\text{MAX}}}\right), \quad W_{\text{MAX}} = 0.9$$

物理含义：突触权重越接近上限（W_MAX = 0.9，对应AMPA受体饱和阈），增益越小，避免权重无界增长。实验五v12实测 `w_mean_final ≈ 1.0`（归一化后），学习阶段末期权重有效聚集在0.85~0.90区间。

**电路原语：乘法累加器（MAC）+ 饱和映射表（LUT）**

```
┌────────────────────────────────────────────────────────────────┐
│  Soft-Saturating STDP Circuit                                  │
│                                                                │
│  输入：                                                        │
│    dw_raw : STDP原始增量（指数衰减内核，符号位区分LTP/LTD）    │
│    w_cur  : 当前权重（6-bit定点，0~63，精度≈0.014/LSB）        │
│                                                                │
│  软饱和计算（LUT方案，硬件友好）：                              │
│    预存64项饱和因子表 sat_lut[w] = (64-w) >> 6                │
│    输入w_cur → 查表得因子 sat_factor                           │
│    dw_eff = dw_raw × sat_factor  (移位乘法)                   │
│    64-entry × 6-bit LUT，面积 ~2 logic cells                  │
│                                                                │
│  乘法方案（精确）：                                             │
│    ┌──────────┐   ┌──────────┐   ┌──────────────────┐        │
│    │  w_cur   │──►│ 64-w_cur │──►│  6×6 乘法器      │        │
│    │ 6-bit    │   │ 减法器   │   │  积:12-bit→6-bit  │◄─dw   │
│    └──────────┘   └──────────┘   └──────────────────┘        │
│                                                                │
│  面积估算（28nm工艺）：                                        │
│    LUT方案：~15 logic cells/bond（推荐）                       │
│    乘法器：~40 logic cells/bond                                │
│  时序：1个时钟周期完成软饱和计算（@100MHz → 10ns）             │
└────────────────────────────────────────────────────────────────┘
```

**权重寄存器宽度需求：**

| 参数 | 值 | 硬件映射 |
|------|-----|----------|
| W_MAX | 0.9 | 6-bit定点：0~63，精度 ≈ 0.9/63 = 0.014/LSB |
| 有效上限 | w_cur = 57（≈0.9×63） | 锁存器：w_cur > 57时拒绝继续写入 |
| 软饱和区间 | w_cur > 32（>0.5×W_MAX） | 触发LUT软饱和模式，切换衰减增益路径 |

#### 7.1.2 稳态权重归一化（Turrigiano 2012 Nat Rev Neurosci）

**数学定义**

$$w_i^{\text{new}} = w_i^{\text{old}} \times \frac{\text{TARGET}}{\bar{w}}, \quad \text{TARGET\_W\_MEAN} = 0.50$$

其中 $\bar{w}$ 为学习阶段结束后全网络权重均值。参数 TARGET_W_MEAN = 0.50 由v5成功参数反推，对应有效传播概率 $P_{\text{prop}} \approx 0.17$（平均激活率 ≈ 17%）。

**电路原语：树形加法器（全局均值）+ 广播乘法单元**

```
┌────────────────────────────────────────────────────────────────┐
│  Global Weight Normalization Engine                            │
│                                                                │
│  阶段1：全局权重均值计算（树形加法器）                          │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │  Level 0: N×K 个6-bit权重寄存器（并行读出）               │ │
│  │  Level 1: 成对求和 → N×K/2 个7-bit累积器                 │ │
│  │  Level 2: 成对求和 → N×K/4 个8-bit累积器                 │ │
│  │  ...（log₂(N×K) 级流水线共14级）...                      │ │
│  │  Output: SUM（24-bit，防溢出） → 均值 w̄ = SUM/(N×K)     │ │
│  └──────────────────────────────────────────────────────────┘ │
│                                                                │
│  阶段2：归一化因子 adj = TARGET / w̄ = 0.50 / w̄              │
│    8-bit定点除法器，精度 ≤ 1%（硬件设计指标）                 │
│                                                                │
│  阶段3：广播乘法                                               │
│    adj寄存器 → 全局总线 → N×K 个MAC单元                       │
│    流水线：1 bond/cycle，N=279×K=16：延迟≈44.6μs@100MHz      │
│    原子性保障：归一化期间冻结STDP更新（写冲突防护）             │
│                                                                │
│  面积估算：                                                    │
│    树形加法器：~200 logic cells                                │
│    8-bit除法器：~80 logic cells                                │
│    广播乘法阵列（N×K个）：~22K logic cells                     │
│    控制FSM：~20 logic cells                                    │
└────────────────────────────────────────────────────────────────┘
```

#### 7.1.3 BTW极慢驱动（Bak-Tang-Wiesenfeld 1987）

**数学定义**

每 BTW_INTERVAL = 10 个时间步，向网络随机注入一个种子脉冲（激活随机节点），模拟沙堆模型"沙粒添加"操作，维持系统持续处于临界态。

**电路原语：LFSR（线性反馈移位寄存器）+ 计数器**

```
┌────────────────────────────────────────────────────────────────┐
│  BTW Slow Drive Unit                                           │
│                                                                │
│  组件1：16-bit Galois LFSR（低速随机数发生器）                 │
│    多项式：x^16 + x^14 + x^13 + x^11 + 1（最大周期2^16-1）   │
│    时钟：步时钟（1MHz仿真步频），非芯片基础时钟                 │
│                                                                │
│  组件2：4-bit BTW定时器计数器（模BTW_INTERVAL = 10）           │
│    每步+1，到10时：触发trigger_btw、复位、采样seed_id          │
│    seed_id = lfsr_out % N（N=2^k时用掩码，精确高效）           │
│                                                                │
│  组件3：种子注入控制                                           │
│    if trigger_btw: 强制激活神经元seed_id（绕过阈值，单脉冲）   │
│    优先级：低于STDP，高于正常激活逻辑                           │
│                                                                │
│  HDL伪代码（Verilog风格）：                                    │
│                                                                │
│  reg [3:0]  btw_cnt;                                          │
│  reg [15:0] lfsr_btw;                                         │
│  wire btw_fb = lfsr_btw[15]^lfsr_btw[13]                     │
│              ^lfsr_btw[12]^lfsr_btw[10];                      │
│                                                                │
│  always @(posedge step_clk) begin                             │
│    lfsr_btw   <= {lfsr_btw[14:0], btw_fb};                   │
│    btw_cnt    <= (btw_cnt==9) ? 4'd0 : btw_cnt+1;           │
│    btw_trigger<= (btw_cnt==0);                                │
│    btw_seed   <= lfsr_btw[8:0] % N; // 掩码取模              │
│  end                                                          │
│                                                                │
│  面积：~30 logic cells    功耗：<0.1mW（极低）                │
└────────────────────────────────────────────────────────────────┘
```

#### 7.1.4 双重临界点校准（G_kappa + G_size）

**数学定义**

双步校准（实验五v12验证有效，G_kappa = 2.0005，kappa_calib = 0.9706，mean_size_calib = 47.73）：

$$\text{步骤1：}\kappa = \sum_{t} \frac{A(t+1)}{A(t)+\epsilon} \div T, \quad \text{二分法调整 } G \text{ 使 } \kappa \in [0.9, 1.1]$$

$$\text{步骤2：mean\_size} = \frac{\sum_j s_j}{n_{\text{avalanche}}}, \quad \text{微调 } G \text{ 使 mean\_size} \in [5, 80]$$

**电路原语：在线κ估计器 + G全局增益寄存器**

```
┌────────────────────────────────────────────────────────────────┐
│  Online Critical Calibration Unit                              │
│                                                                │
│  子模块1：在线κ估计器（滑动窗口100步）                         │
│    FIFO(100 × 10-bit)：存储近100步激活节点数 A(t)             │
│    每步计算：r_t = A(t+1) × INV_LUT[A(t)]（查表求逆）        │
│    EMA滤波：kappa_est = 0.99×kappa_est + 0.01×r_t            │
│    输出：8-bit定点κ（0~2.56，精度0.01）                       │
│                                                                │
│  子模块2：G全局增益寄存器（16-bit可写）                         │
│    广播：G → 全局总线 → 每个突触的传播权重乘法器              │
│    片外更新（二分法控制器，软件或MCU实现）：                   │
│      kappa < 0.9 → G += ΔG                                   │
│      kappa > 1.1 → G -= ΔG                                   │
│    ΔG步长可编程（实验五v12：ΔG ≈ 0.01）                       │
│                                                                │
│  校准流程：                                                    │
│    运行500步→采样κ→二分法调整G→重复→κ∈[0.9,1.1]收敛         │
│    验证mean_size∈[5,80]→微调G（步长0.001）                    │
│    总校准时间：~2000步 × 1μs/步 = 2ms                         │
└────────────────────────────────────────────────────────────────┘
```

## 第八部分：真实C.elegans连接组的硬件锚定结构

> **实验六来源**：基于Varshney 2011真实C.elegans化学突触+电突触连接组（N=279，connectome=Varshney2011_real），三规则体系下SOC动力学仿真，验证真实connectome的硬件约束。

### 8.1 电突触→E-L键的硬件实现

实验六发现，真实connectome包含 **997条电突触（gap junction）**，作为E-L固化键锚定基础拓扑，初始化时 `n_el = 997`，对应双向 = **1994个固化连接方向**。这些电突触整个学习过程保持固化（不参与STDP），分支比验证值 `branching_ratio = 1.057`（在SOC临界带[0.9,1.1]内），证明固化键不破坏临界态。

**硬件含义：需要区分可塑突触（SRAM可写）和固化突触（ROM只读）**

| τ幂律指数（时长） | **1.712** | **1.667** | 片外统计分析 |
| PSD斜率 | **-1.431** | **-1.533** | 实时功率谱，目标 -1.0 ~ -1.5 |
| 权重上限W_MAX | 0.9 | 0.9（w_mean=0.90） | 6-bit定点，有效上限57/63 ≈ 0.90 |
| 归一化目标 | TARGET = 0.50 | w_mean ≈ 0.90（学习后未归一化） | 全局均值电路精度 ≤ 1% |
| 固化键数量 | 0（纯软件connectome） | **997条电突触** | OTP ROM：~32Kbit（C.elegans） |
| 零帧比例 | **31.7%** | **39.2%** | 动态电源门控阈值：活跃率 < 60% |
| 平均雪崩尺寸 | **56.63节点/雪崩** | **57.52节点/雪崩** | 监测窗口：每1000步统计一次 |

**关键设计结论：**
1. 实验五与实验六κ值高度一致（1.061 vs 1.057），证明SOC临界态对连接组结构具有鲁棒性，硬件κ监测精度±0.05即可覆盖两种实现模式
2. 实验六零帧比例（39.2% > 31.7%）更高，真实connectome固化拓扑天然更节能——固化的电突触骨架提供更高效稀疏激活模式，动态功耗进一步降低约7.5%

## 附录D：四规则体系 vs 三规则体系硬件开销对比

| 原语模块 | 三规则（原版） | 四规则（实验五扩展） | 增量 |
|---------|-------------|------------------|------|
| STDP Bond | 36-bit/bond（硬截断w上限） | 36-bit/bond + LUT软饱和（~15 cells额外） | +15 logic cells/bond |
| 全局缩放引擎 | 激活率EMA + 本地乘法器 | 树形加法器 + 广播乘法 + 除法器 | +~22K logic cells |
| 随机驱动 | 32-bit LFSR（WS重连） | 复用LFSR + 新增4-bit BTW计数器 | +30 logic cells |
| 临界监测 | 无 | 在线κ估计器（FIFO+EMA+G寄存器） | +~600 logic cells |
| 突触存储分区 | 单一SRAM | SRAM（可塑）+ OTP ROM（固化） | +~32Kbit OTP硬宏 |
| 互连结构 | 单层可重构crossbar | 三级流水线分区路由 | +~5K logic cells |
| **总面积** | **~5.0mm²** | **~5.6mm²** | **+12%面积** |
| **总功耗（活跃）** | **~81mW** | **~90mW**（新增逻辑开销） | **+11%功耗** |
| **总功耗（空闲）** | **~18mW** | **~12mW**（零帧门控增强） | **-33%空闲功耗** |

**结论**：四规则体系在三规则基础上仅增加约**12%面积、11%活跃功耗**，同时带来：
1. SOC动力学在线精确校准（κ±5%精度，收敛<2ms）
2. 真实connectome固化锚定结构支持（OTP ROM，997条电突触）
3. BTW极慢驱动（<0.1mW，维持临界态长期稳定）
4. 空闲功耗降低33%（零帧门控从31.7%→39.2%，真实connectome结构天然节能）

**四规则体系是SDSoW从"可工作原型"到"可量产神经形态SOC"的工程关键跨越。**
