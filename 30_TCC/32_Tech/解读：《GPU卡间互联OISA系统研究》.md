**大模型发展与集群规模变化：**从 GPT3 到 GPT6 阶段的发展过程，其参数级别从 0.175TB 逐渐增长到预估的 100TB，从 2020 年到 2028 年，计算集群的规模也从小集群发展到了 “百万” 卡集群。这一系列的变化都表明了人工智能领域对计算资源需求的急剧增长。

**大模型是演进的方向：**大模型正朝着超万亿参数、多模态、长序列的方向发展。从千亿到超万亿参数，从短序列到长序列，从单模态向多模态转变，这些变化为模型带来了诸多优势。例如，集成多种数据类型使得模型能够同时处理文本、图像、声音等多种数据类型，从而更全面地理解情境，提高决策和响应的准确性；长序列能够提供更多的上下文信息，更准确地理解和预测数据模式，进而处理更复杂的任务，如长文本生成、长期时间序列预测等。同时，GPU 计算能力的进步也为训练和运行这些更大的模型提供了可能，并且随着模型参数量的增加，GPU 资源也能得到更充分的利用。

**模型训练与 GPU 效率的关系：**为了更深入地理解 GPU 在模型训练中的作用，我们来看一下 GPT - 3 模型的训练情况。从这张幻灯片中我们可以看到，在 GPT3 等千亿模型规模的条件下，随着 GPU 数量的增加，训练时间产生近乎线性的扩展。这意味着更多的 GPU 可以在一定程度上缩短训练时间。同时，我们还发现具有更大的模型比较小的模型可以更有效地训练，而且算力增加可缩短训练时间，参数量增加更能充分利用 GPU 资源。这里通过两个图表为大家详细展示，一个是 GPT - 3 模型的训练时间与 GPU 数的函数关系，另一个是 A100 矩阵乘法性能与矩阵大小的函数关系。

**算力底座新的跨域式发展：**随着智算需求的激增，数据中心在 “存算网管效” 五大方面都需要全面升级，从而完成向 NICC 新型智算中心的跃迁。我们可以看到，万亿以上的模型驱动智算底座迎来了 Mega 时代，这需要第二次跃迁，也就是呼唤科学技术和工程实施的融合创新。AI服务器由单机 8 卡向整机柜百卡互联方案演进。

**张量并行GPU数量决定AI集群效率：**单机 8 卡内，我们通常采用 Switch 或 Full mesh 网络，在这种情况下，张量并行需要充分利用机内的带宽能力。我们知道，算力规模相对大模型效率会存在一个 “甜点区”，我们的目的就是要扩大这个 “甜点区” 的面积。张量并行的GPU数量会影响运算效率，如果张量并行的GPU数量较少，内网络带宽没有充分被利用，如果张量并行需要通过机间网络通信，则通信效率降低，影响训练效率。

**国内 GPU 机内互联拓扑结构：**主要有两种典型的拓扑结构，分别是 Cube Mesh 和 Full Connection，这两种拓扑都存在一定的局限性；

**超万亿模型的需求与挑战：**随着模型朝着超万亿参数发展，我们面临着新的需求和挑战。超万亿参数的模型需要超节点来支持，因为张量并行、专家并行流量会激增，传统的 8 卡服务器在训练效率上存在互联短板。而且由于芯片 Ports 数量有限，GPU 卡间互联性能需要进行革新，才能满足超节点形态服务器的研发条件。在这个过程中，我们可以看到，从千亿到万亿再到超万亿，卡间流量的情况也在发生变化，卡间（百卡）流量激增，而机间流量占比降低。这一切都表明 GPU 卡间互联成为了关键要素，我们需要重塑技术能力和生态环境。

**GPU机内互联从8卡形态向超节点形态发展：**张量、专家并行等通信需求引发智算设施再次升级，从单机 8 卡向超节点演进，GPU 卡间互联技术成为关键。我们发现 GPU 直连无法持续满足需求，引入交换芯片是主要的确定性路线，但产业生态是最大的难题。目前产业界迫切需要一个标准化的解决方案，以实现 GPU 从 8 卡形态走向超节点。

**全向智感互联OISA 技术体系：**全向智感互联 “OISA” 开放 GPU 互联技术体系。它是一种大规模 GPU 对等互联的技术，能够保障高效全连接路径。我们通过重塑报文格式，提高了载荷率，并且通过动态感知，提升了互联效率。同时，我们还重新制定了数据层可靠性保障机制，采用选择性重传，提高了恢复效率。在物理传输方面优化了 SerDes、控制器等物理层组件，并引入了 CPO/NPO，支持光电融合技术。

**GPU卡间带宽、时延、无损是OISA 核心内涵**：在带宽方面，将当前 P2P 带宽能力从约 50G/s 跃升到 800GB/s；在时延方面，从跨机数微秒级甚至毫秒级提升到超节点内纳秒级别；在无损方面，拥塞后希望快速感知，避免丢包，减少流控，快速爬坡，实现感知（无丢包）的目标。

**OISA 协议三层架构设计：**为了实现 OISA 技术体系的目标，我们设计了 OISA 协议，它分为三层：事务层（TL）、数据层（DL）、物理层（PL）。
- 
事务层（TL）主要负责生成和接收事务报文，用于执行 GPU 读写原语操作的报文，处理端到端事务，管理地址转换；
- 
数据层（DL）负责数据包的可靠性传输，流量控制，避免数据溢出或丢失，数据包的排序和重组；
- 
物理层（PL）负责通过物理媒介传输原始比特流，定义了电气特性、信号完整性、连接器类型以及编码和解码。

**OISA - Switch 实现集合通信加速：**与传统多次归约计算不同，使用 “OISA - Switch” 更有效利用 Switch 的集合通信加速能力，降低计算和通信的规模，提升模型处理效率。我们通过 Reduce Scatter、Gather All 等操作来展示它的优势。比如在这些操作中，传统方式可能需要进行多次归约计算，而使用 OISA - Switch 则可以更有效地利用 Switch 的集合通信加速能力，减少计算和通信的工作量，从而提高模型处理效率。

**OISA 技术体系的目标：**OISA 技术体系的目标是为 GPU 互联领域建立一个开放、高效、统一且低成本的技术体系，支持 GPU 产业的繁荣发展。在超节点内部引入 Scale Up 的 OISA Switch 芯片，可以实现 P2P 性能提升。OISA Gen1 设计支持满足 128 张 GPU 芯片的高效互联、P2P 卡间带宽 800GB/s、Switch 芯片最大 I/ 0 带宽 51.2T。

**建立OISA 协同创新平台：**为了推动 OISA 技术和产业的发展，中国移动、北京市科委、中关村管委会携手 48 家企业、高校和科研机构，建立了 “OISA 协同创新平台”。通过这个平台，我们共同推进 OISA 卡间互联技术和产业的发展。目前，OISA 协议的架构设计、报文优化、拥塞控制和重传机制已初步完成，并且已经发布了芯片原型。我们希望通过这个平台，凝聚各方力量，打造 “北京全向智感 OISA 协同创新平台”，构建 OISA 协议体系，研发 OISA 硬核 IP，开发 OISA 芯片和超节点，实现标准、研发、应用的闭环，持续吸纳更多伙伴加入。

**OISA 超节点服务器概念设计**：在 OISA 技术体系的应用方面，我们开展了 OISA 超节点服务器的硬件设计，并制定了硬件设计规范。OISA 超节点服务器集群架构采用灵活配置、模块化、标准化设计，统一各部件之间接口，支持根据机房条件、业务需求灵活配置算力资源规模。同时，节点内采用全电连接，具有高可靠性，低功耗，采用冷板式液冷，集中散热，集中供电，提高密度降低 PUE。此外，我们还探索通算、智算、交换资源解耦池化，支持多芯片多架构，以实现异构解耦资源池化。
