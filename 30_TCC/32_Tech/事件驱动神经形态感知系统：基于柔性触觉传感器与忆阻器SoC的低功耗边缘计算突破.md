# 事件驱动神经形态感知系统：基于柔性触觉传感器与忆阻器SoC的低功耗边缘计算突破

# 背景介绍

* 受 IoT 与机器人扩展驱动，传感节点与产生的数据爆炸性增长。
* 传统做法：模拟信号 → ADC → 数字处理，造成大量无效数据、高能耗与高延迟。
* 生物感知是“事件驱动”的：只有变化才发信号。本文目标是把这套思想用硬件全程模拟实现，尽量消除 ADC/数字搬运开销。

![图片](https://get-notes.umiwi.com/morphling%2Fvoicenotes%2Fprod%2F010b919bc77fd0f6ac1681db09012c8a?Expires=1780059710&OSSAccessKeyId=LTAI5t7toTp72R3TvdXf9QdK&Signature=1DjRtZcz1kqk5OlkxeLeXaUUoAs%3D)

图 1｜事件驱动神经形态感知系统总体架构（概览）

a, 柔性压电触觉传感器阵列示意图：由预极化 PVDF 薄膜、顶/底电极（Au）与 PDMS 封装组成，当发生动态变形（按下/释放）时产生异步尖脉冲。

b, 传统帧式数字感知流水线：模拟信号经放大/滤波后被 ADC 连续采样，数字数据存于内存并由 CPU 处理，导致持续采样与数据搬运开销。

c, 传统冯·诺依曼式数字计算示意：存储与计算分离产生数据搬运瓶颈。

d, 事件触发预处理电路示意：仅在像素事件发生时激活对应通道，脉冲触发模拟开关并为 RC 电容充电，随后以预设时间常数衰减，生成用于构造 time surface 的模拟波形；只有发生事件的通道处于活跃态，增强数据稀疏性与能效。

e, 忆阻器 SoC（TetraMem MX100）与系统集成照片：SoC 包含多个 VMM 计算核（248×256 交叉阵列）、RISC-V CPU 与必要外围电路，支持将 time surface 的模拟向量映射到阵列并在模拟域完成聚类与分类推理。

# 一句话解释

马萨诸塞大学夏强飞团队提出了一套端到端事件驱动、全模拟的触觉感知系统：由柔性压电传感器阵列触发事件 → 事件触发的 RC 衰减电路把尖脉冲编码成“时间面（time surface）” → 忆阻器（memristor）系统-on-a-chip（SoC）在模拟域完成聚类与分类推理，从而实现显著降低延迟和能耗的边缘识别。

# 系统构成

## 1. 柔性压电触觉传感器阵列（sensor）

材料与结构：预极化 PVDF 薄膜（50 μm）夹在 Au 电极之间并由 PDMS 封装（见 Methods 与 Extended Data）。

特性：对动态压力变化（按下/释放）产生瞬态电压脉冲（持续约几十 ms），自供电（无需持续供电）。释放时可见反向脉冲。对静态恒力不敏感（低频盲区）。

b, 4×4 阵列若干像素的原始输出示例（逐像素电压随时间曲线），显示离散的正向脉冲与释放时的反向脉冲。

## 2. 事件触发预处理电路（one-to-one）

重要澄清：传感器脉冲仅用于触发；衰减起始电平由固定输入端保证，从而使不同事件的残余电压具有可比性。

每像素一对一连接到一个处理单元：仪表放大器 + 模拟开关 + RC（R+C）电路。

工作原理：当传感器输出的正向尖脉冲超阈值时，模拟开关闭合，电容被充到固定电压；脉冲结束后开关断开，电容按预定时间常数指数放电，形成起始幅度一致的衰减波形。时间顺序被编码为残余电压幅值（先发生的事件残余较小，后发生的残余较大）。

## 3. 忆阻器 SoC（TetraMem MX100）

组成：十个 248×256 memristor 交叉阵列核（用于大规模 VMM）、RISC-V CPU、片上 RAM、IO 与外围电路（采样/复用/跨阵列编程等）。

功能：将预处理得到的时间面向量作为行/列电压输入，利用交叉阵列在模拟域并行完成向量-矩阵乘法（VMM）；配合片上 CPU 实现聚类中心更新、SLP（单层感知器）训练与权重编程，最终实现全片上推理。

# 算法与数据流

## 1. 事件发生 → 构建 time surface（3×3 或 5×5）

每次事件只在该像素及其邻域构造 time surface。实验小数组使用 3×3（包含位置信息 x,y + 9 个电压 = 11 维）；扩展数据集使用 5×5。

## 2. 聚类（mini-batch k-means，使用余弦相似度）

聚类中心（features）被归一化并编程到 memristor 阵列（每列对应一个中心）；将 time surface 作为输入电压到 BL（bit-lines），列电流代表相似度。每个 time surface 被分配到相似度最高的簇（列）。

## 3. 统计特征计数 → SLP 分类

对一整条笔画序列的 time surfaces 做计数，得到 feature histogram（向量 C），将其送入另一交叉阵列的 SLP 权重，VMM 输出为分类得分。

实现要点：聚类与 SLP 的权重都被映射为 memristor 的电导值（实验编程精度约 8-bit）。

# 性能实测

## 1. 硬件验证（4×4 触觉阵列）

三字母任务（T、C、L）：基于 time-surface 的硬件推理准确率86.67%（混淆矩阵显示 T 完全识别，C 与 L 有小量混淆）。（见 Fig.3 说明）

若将同一数据用 CNN（基于对衰减信号的单帧“快照”）在 memristive SoC 上实现，实验可达到100%（3 类），说明 frame-based 方法在短时全局信息下能获得更高准确性，但牺牲了事件驱动稀疏性与实时性。

## 2. 规模化验证（仿真并映射到 SoC，32×32）

自制数据集：10 个数字，每数字区分顺/逆时针，共 20 类，每类 375 样本 → 总 7,500 模式；生成约 428,026 个 time surfaces。

time-surface 方法（聚类 15 个特征，5×5 邻域）结果：软件仿真 93.83%／硬件实现（SoC）91.83%。

## 3. 能耗 / 延迟（EDP）比较

作者估算：单次推理的能耗-延迟积（EDP）在 memristive SoC 上14.86 pJ·s，而选择的高性能数字平台（Intel Core i9-14900K + AD9002 ADC）为254.7 pJ·s，约17×改进（估算基于 Supplementary Notes，注意为比较基准选择带有理想化假设）。

# 局限性

压电传感器对静态力/低频不敏感（需异构传感或额外测力模块以覆盖持续握力等用例）。

模拟信号脆弱性：温漂、电磁干扰、长线传输造成噪声与衰减，放大阵列规模时需高质量前端与封装/屏蔽设计。

时间常数固定：RC 决定记忆窗口；写字速度差异会影响 time surface 表征，长期序列或复杂时序任务可能需在芯片端引入具有时间记忆的神经单元（如 RNN/Reservoir）。

系统目前为“采样-批处理”式：实验中的 time surfaces 先存片上内存，再做聚类/分类；要做到真正的逐事件实时流式处理，需要更紧耦合的前端–SoC 协同设计（作者也在 discussion 中提到）。
