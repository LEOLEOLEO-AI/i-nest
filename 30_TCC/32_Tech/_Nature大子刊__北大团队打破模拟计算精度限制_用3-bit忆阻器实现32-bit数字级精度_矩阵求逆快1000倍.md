这里是类脑智能计算，今天带大家解读一篇2025年10月13日发表在 Nature Electronics 上的重磅论文。 论文题目： Precise and scalable analogue matrix equation solving using resistive random-access memory chips
DOI: 10.1038/s41928-025-01477-0 关键词： 模拟矩阵计算(AMC) / 忆阻器(RRAM) / 矩阵求逆 / 迭代细化 / 存内计算

模拟计算（Analogue Computing）利用物理定律（如欧姆定律）在内存中直接进行计算，展现了超高的能效和速度，然而，它一直有一个致命弱点：精度太低。 受限于器件噪声和工艺偏差，模拟计算通常只能达到4-8位精度。在简单的AI推理这个精度还可以，但在科学计算、求解微分方程或者AI训练（需要矩阵求逆）这些高精度应用场景中，就显得力不从心。 如何用“不完美”的模拟器件算出“完美”的精确结果？北京大学黄如院士、蔡一茂教授、孙仲研究员团队给出了一个惊艳的答案。
## 一句话解释
提出了一种基于迭代细化（Iterative Refinement）的混合模拟计算架构，利用40nm工艺的3-bit RRAM芯片，通过“低精度求逆+高精度乘法”的循环校准，成功将模拟矩阵方程求解的精度提升到了24-bit（对标FP32数字精度），在Massive MIMO信号检测中实现了与数字处理器一致的性能，但速度快了三个数量级。

图注解读：

## 核心技术
## 1. 迭代细化算法 (Iterative Refinement)：
作者设计了一个闭环系统：
- 
粗算 (LP-INV)： 先用模拟电路做一个“大概”的矩阵求逆。这步速度快，但有误差。
- 
精修 (HP-MVM)： 计算当前的残差（Residual，即 b - Ax）。为了保证这一步准，作者使用了位切片（Bit-slicing）技术，把高位和低位拆开算，用多个3-bit RRAM单元拼凑出高精度乘法。

图2：硬件实现与收敛曲线
图注解读：
(a) 电路原理： 展示了闭环的LP-INV电路（左，带运放反馈）和开环的HP-MVM电路（右，位切片）。
(b) 芯片实拍： 左边是1Mb的RRAM大阵列用于高精度乘法，右边是8x8的小阵列用于求逆。
(e-f) 收敛过程： 随着迭代次数（Cycle 1, 2, 3）增加，电路输出（红点）迅速逼近理想值（虚线）。误差（柱状图）呈指数级下降，3个周期后就极其微小了。
2. BlockAMC
单片RRAM阵列大小有限，怎么算大矩阵？
- 
分块策略： 作者提出了BlockAMC算法，将大矩阵切分成小的子块（Block），分发到不同的RRAM阵列上计算，最后再拼装起来。这解决了模拟计算难以扩展（Scalability）的问题。

图3：复杂任务扩展——复数与大矩阵
图注解读：
(c) BlockAMC流程： 像拼积木一样，把大矩阵切成M1-M4四个小块，分别调用求逆（INV）和乘法（MVM）操作。
(e) 复数矩阵： 处理复数矩阵（通信中常见）时，将其展开为实数矩阵。实验显示，10次迭代后误差降至 10^-7。
(g) 16x16矩阵求解： 这是目前模拟电路求解的最大规模之一。红点（实验值）在10次迭代后与蓝线（理论值）完美重合，证明了扩展性的成功。
## 3. 硬件实测：40nm RRAM芯片
- 
这不是纯仿真。团队真的流片了40nm TaOx RRAM芯片（1T1R结构），并实现了稳定的3-bit（8个电导等级）编程，良率100%。
## 性能实测
精度：在求解16x16矩阵方程时，经过10次迭代，相对误差降到了 10^（-7） 量级，相当于24-bit的定点精度，完全可以媲美32位浮点数（FP32）。
应用：在5G/6G的关键技术——Massive MIMO信号检测中，该系统仅需3次迭代就能达到和数字FP32处理器完全一样的误码率（BER），证明了它能处理复杂的通信任务。
超高能效：基准测试显示，在同等精度下，该方案的吞吐量（Throughput）比GPU（Nvidia H100）和专用ASIC高出1000倍，能效（Energy Efficiency）高出100倍。

## 讨论
## Q1: 既然用了数字控制和DAC/ADC，这还算“模拟计算”吗？
算，这是数模混合（Mixed-Signal）架构。核心的矩阵运算（也是最耗能、最慢的部分）是在RRAM阵列中以模拟电流形式完成的（O(1)复杂度）。数字部分仅负责轻量级的控制和残差更新。
## Q2: 为什么3-bit的器件能算出24-bit的结果？
这就是迭代的魔力。这就像你在称重，先用大砝码（低精度）测个大概，再用小砝码（高精度）测剩下的零头。虽然每次测量都有误差，但只要误差在一定范围内，通过不断的“测量-修正”，最终结果的精度只取决于你愿意迭代多少次（以及噪声底限）。
## Q3: 这个技术对AI有什么用？
目前的存内计算（CIM）大多只能做推理（Forward）。因为训练（Training）需要计算梯度和更新权重，这往往涉及二阶优化（如牛顿法），需要矩阵求逆。这项技术让RRAM芯片不仅能“推理”，未来还有可能直接在端侧进行“训练”和“微调”。
## 局限性
1. 模拟求逆的扩展问题
作者坦诚，相比于成熟的矩阵乘法（MVM），构建大规模的模拟求逆（INV）电路要难得多。
- 
连线：闭环反馈电路需要复杂的物理连线，这在芯片版图设计上会带来巨大的路由压力。
- 
电流：求逆时阵列所有行列同时激活，会导致位线/源线上产生巨大电流，引发严重的IR-drop，直接影响计算准确性。目前演示的核心仅为8*8，要做大需要极其精细的电路设计。
2. ADC的能耗限制
为了实现高精度迭代，模拟域和数字域之间需要频繁的数据交换。论文分析指出，整个系统的芯片面积其实是被ADC（模数转换器）主导的，而不是RRAM阵列本身。这导致在小矩阵下，面积优势并不明显。
