# 统一内存与网络语义架构（Risc-V结合）

---

# 项目指南 C：统一内存与网络语义架构 (Project: OneFabric-Memory) ——谢涛

**核心目标：** 消除“内存访问”与“网络通信”的软件边界，使分布式集群在逻辑上表现为一块巨大的共享 SRAM 池。

1\. 关键技术模块

-   **全局统一寻址空间 (Global Unified Address Space, GUAS)**：
    
    -   **设计**：将 64 位地址空间划分为 `[机架 ID | 芯片 ID | 核心 ID | 本地偏移]`。
        
    -   **效果**：`Load/Store` 指令可以像操作本地寄存器一样操作 100 米外的另一颗芯片的 SRAM。
        
-   **硬件级直接远程访问 (H-DRMA)**：
    
    -   **不同于 RDMA**：完全绕过 CPU 和操作系统内核，由 NoC 硬件直接解析远程地址。
        
    -   **实现**：当核心 A 访问远程地址时，NoC 自动将其转化为一个以太网请求包，目标核心的 NoC 硬件直接从 SRAM 返回数据，全过程**无软件干预**。
        
-   **时延隐藏缓冲区 (Latency-Hiding Prefetcher)**：由于远程访问延迟远高于本地，需设计一套基于数据流预取的硬件引擎，预测下一组需要跨网络搬运的内存块。
    

2\. 实施路径

-   **第一阶段（内存模型定义）**：定义弱一致性内存模型（Weak Consistency），允许 AI 计算在数据尚未完全同步时进行预测性计算，以容忍网络延迟。
    
-   **第二阶段（硬件网关重构）**：在 Tenstorrent 的 Wormhole 芯片基础上，重写其以太网控制器的微码，支持直接物理地址映射（Direct Physical Mapping）。
    

3\. 核心指标 (KPI)

-   **编程复杂度**：减少 90% 的手工通信代码（如 NCCL `all_reduce`）。
    
-   **访问延迟**：跨芯片 SRAM 访问延迟压缩至 200ns - 500ns 级别（取决于物理链路）。
