---

# 1. 先定义：网络直径是什么？

**网络直径 = 网络中最远两个节点之间的最少跳数**

跳数越少，延迟越低、性能越好。

- N = 节点总数
- 直径越接近常数越好
- 直径越大，扩展性越差

---

# 2. 三种复杂度的直观理解（超级重要）

## ① O (N) —— 最差，线性增长

**典型拓扑：线性链 Chain**

plaintext

```
●—●—●—●—●—●—●—●—●—●—●—●
```

- 两头节点通信要走 **N-1 跳**
- 节点变多，跳数**线性变长**
- 大规模直接废掉

## ② O (√N) —— 中等，慢一点

**典型拓扑：2D Mesh**

plaintext

```
●—●—●
| | |
●—●—●
| | |
●—●—●
```

- 边长 k = √N
- 最远两点：**k 跳 = O (√N)**
- 比线性好很多，但大规模依旧膨胀

## ③ O (logN) —— 优秀，几乎不怎么涨

**典型拓扑：HyperCube、Fat-Tree、Butterfly、Dragonfly**

plaintext

```
      ●——●
     /|  /|
    ●——● |
    | | | |
    | ●—|—●
    |/  |/
    ●——●
```

- 节点翻一倍，跳数**只加 1**
- N=1024，log₂N=10 跳
- N=1e6，跳数也才 20 左右
- **大规模超级能打**

| O(√N)   | 2D Mesh                      | 32 跳   | 128 跳   | 1000 跳 | 一般  |

---

# 4. 用一句话记住

- **O(N)**：节点越多，路越长 → **废拓扑**
- **O(√N)**：路变长，但慢一点 → **普通拓扑**
- **O(logN)**：节点暴增，路几乎不变 → **顶级拓扑**

---

# 5. 对应到你研究里的意义

- **Mesh、Torus**：O (√N) → 小规模能用，大规模瓶颈
- **HyperCube、Fat-Tree、Dragonfly、Butterfly**：O(logN) → **AI / 超算真正能用的高性能拓扑**
- **你的 SDI**：用 P2P+Star+Ring 元拓扑递归，**能直接构造 O (logN) 拓扑**，这就是核心价值
