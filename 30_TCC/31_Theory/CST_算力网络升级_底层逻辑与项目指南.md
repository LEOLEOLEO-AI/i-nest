## 第一部分：底层逻辑——为什么CST思想能让算力网络获得1+1>2

### 1.1 当前算力网络的根本困境

现有大模型算力网络（胖树/Clos/Dragonfly/全交换）的设计哲学是：

```
目标：连通性最大化 → 任意节点对之间的带宽均等
手段：静态拓扑骨架（流量矩阵基于训练任务预先规划）
代价：骨架刚性（网络拓扑固定后，面对多任务时资源利用率急剧下降）
```

**关键数据**：

| 指标 | 实测状况 | 根因 |
|------|---------|------|
| GPU利用率（训练）| 40-60%（MFU） | 通信等待，拓扑刚性 |
| 多任务能效损失 | 30-50%额外开销 | 任务切换需要重新路由 |
| 算力叠加效率 | 97%线性（NVL72实测） | 超线性潜力未被释放 |
| 跨任务迁移 | 几乎为零 | DSA骨架针对特定模型 |

**本质诊断**：现有算力网络是为"算力堆砌"设计的，追求的是让亚线性叠加逼近线性（σ_branch→1但不超过）。这正好是SOC临界态以下——次临界区间，动态范围被压缩到最低。

### 1.2 CST介入的物理逻辑

**核心洞察**：大模型推理/训练过程中，算力节点之间的激活模式（activation pattern）本质上就是一种"网络雪崩"——信息从注意力层→FFN层→层间传播，与神经雪崩具有相同的数学结构。

```
传统网络视角：
  GPU集群 = 算力节点 + 静态互连
  优化目标 = 带宽最大化 + 延迟最小化

CST视角（革命性重构）：
  GPU集群 = 计算节点网络
  互连网络 = 动态可塑骨架（E-L/E-S化合键等价物）
  优化目标 = 网络时空复杂度CST最大化
              ↓
              临界态（τ→1.5）
              ↓
              动态范围×10倍 + 任务多样性×N + 跨任务迁移
```

**物理机制三步推导**：

**第一步：算力网络的"突触"等价**

| 生物神经网络 | 算力互连网络 | 对应关系 |
|------------|------------|---------|
| 神经元激活 | GPU/TPU算力单元激活 | 计算节点 |
| 化学突触（强度可变）| 光纤/铜缆带宽（可调度）| 动态带宽 |
| 电突触（固定快速）| InfiniBand/NVLink骨干 | 低延迟骨干 |
| E-L（固化骨架）| 训练期固定热路径 | 高频通信路径 |
| E-S（可塑学习）| 推理期动态调度路径 | 低频长尾路径 |
| STDP可塑性 | 流量自适应路由（动态权重）| 路由可塑性 |
| 突触缩放 | 带宽资源弹性伸缩 | 资源弹性 |

**第二步：为什么现有网络卡在次临界**

现有网络设计追求"无阻塞"（non-blocking），即任意流量矩阵都能无损传输。这等价于：

```
σ_branch ≤ 1（每个激活节点触发≤1个下游节点）
```

这是次临界的定义。次临界网络的动态范围最小（Shew 2009），意味着：
- 对微弱信号（小任务）：放大不足，资源闲置
- 对强信号（大任务）：饱和截止，无法扩展

**第三步：CST可塑骨架如何突破线性上限**

引入CST思想后，网络拓扑不再固定，而是根据任务流量自适应演化：

```
任务到来 → 触发"网络雪崩"（激活路径级联扩散）
雪崩大小S服从幂律P(S)∝S^(-3/2)（临界态）

效果：
  小任务 → 小雪崩（高效，不浪费资源）
  大任务 → 大雪崩（自发扩展，调用更多节点）
  跨任务 → 骨架E-L热路径保留，新任务走E-S可塑路径
```

这就是**系统级1+1>2的物理来源**：不是更多GPU，而是网络动态范围扩大10倍（Shew 2009），使得系统在J/task维度的效率非线性提升。

### 1.3 三个维度的超线性增益来源

**维度一：能效比 E_task↓（J/task）**

```
机制：临界态网络动态范围×10倍（Shew 2009）
      → 小任务用小子网完成，无需激活全局骨架
      → 等效于：同等任务消耗能量降至1/10
实测类比：人脑20W完成语言理解 vs GPT推理~3MW
          差距15万倍，来源正是拓扑动态范围差异
```

**维度二：任务多样性 D_task↑**

```
机制：E-L骨架固化训练热路径 + E-S可塑路径动态适配新任务
      → 同一硬件同时支持：训练大模型（E-L骨架）
                         + 推理多样任务（E-S动态路径）
                         + 细粒度微调（E-S局部调整）
      → 任务类型多样性从"1种DSA"扩展到"N种动态适配"
```

**维度三：跨任务迁移速度 V_transfer↑**

```
机制：类比STDP（Bi&Poo 1998）的突触时序规则
      → 两个任务共享激活路径时，路径权重增强（LTP等价）
      → 新任务沿已有热路径快速传播（迁移速度↑）
      → 对应生物：睡眠期突触巩固，白天快速联想迁移
实现：流量调度器记录任务对"共激活"统计，类比STDP规则
```

### 1.4 与现有技术路线的关键差异

| 维度 | 现有路线（胖树/Clos/Dragonfly） | CST可塑网络 |
|------|-------------------------------|------------|
| 拓扑设计哲学 | 静态均等连通 | 动态时空复杂度最大化 |
| 优化目标 | 带宽×延迟 | CST = Sc × Tc × e^(α·Γst) |
| 扩展方式 | Scale up（更大骨架） | Scale out（自适应拓扑演化） |
| 任务适配 | 人工重规划路由 | 自组织（STDP等价规则） |
| 能效模型 | J/FLOPS（计算效率） | J/task（任务完成效率） |
| 上限 | 亚线性叠加逼近线性 | **超线性（临界态，动态范围×10）** |
| 未来路线 | 更多光纤，更大交换机 | **软件定义可塑互连（SDI-Net）** |

## 第二部分：项目指南

### 【网络领域】项目指南

---

**项目名称**：面向多任务算力协同的自适应互连网络拓扑生成与动态调控方法研究

**领域分类**：网络与通信 / 高性能计算网络 / 下一代数据中心网络

**核心问题**：
现有大规模算力网络（胖树、Dragonfly、全光交换）采用静态拓扑骨架，面向单一大模型训练场景设计，在多任务并发、任务动态切换场景下，系统整体能效比（J/task）低于理论最优值的30%，且缺乏跨任务资源复用的自组织能力。

**主要研究内容**：

**任务一：多时间尺度流量模式的分形特征量化方法**
- 建立算力网络流量的时间复杂度Tc量化模型（临界性/同步性/可塑性/多时间尺度）
- 研究大模型推理/训练流量的幂律分布特征（是否存在P(S)∝S^(-τ)）
- 建立流量"雪崩"事件检测与临界性评估方法
- 预期指标：流量幂律指数τ的测量精度≤5%，Tc量化模型覆盖时间尺度≥4个数量级

**任务二：基于时序相关性的自适应路由权重调控机制**
- 研究类STDP的流量调度规则：两节点对"共激活"次数超过阈值时，路径权重增强
- 建立路径权重的长时程增强（LTP等价）与长时程抑制（LTD等价）动态平衡机制
- 研究路径"固化-衰减"时间常数与任务迁移速度的量化关系
- 预期指标：跨任务路由收敛时间相比静态路由缩短≥40%

**任务三：网络动态范围与任务多样性的协同优化理论**
- 建立网络拓扑小世界系数σ与任务并发数之间的量化关系
- 研究骨架路径（E-L等价）与可塑路径（E-S等价）的最优比例（目标：骨架约20%）
- 建立系统级超非线性增益IE三元指标（E_task/D_task/V_transfer）的测量方法
- 预期指标：多任务场景下J/task降低≥30%，任务多样性支持数量提升≥3倍

**任务四：面向临界态的网络自组织调控软件系统**
- 实现基于分支过程临界性检测的实时网络状态评估模块
- 研究网络从次临界→临界态的主动调控策略（类比突触缩放机制）
- 开发原型系统并在GPU集群（≥256节点）上进行验证
- 预期指标：系统态流量幂律指数τ达到[1.5,2.5]，GPU利用率提升≥15%

**预期重大成果**：
- 提出"算力互连网络时空复杂度"的量化理论框架（1项）
- 形成自适应路由调控软件系统原型（1套）
- 发表顶级网络/计算机体系结构期刊论文≥6篇（SIGCOMM/NSDI/ToN级别）
- 申请发明专利≥8项

**建议经费**：3000-5000万元（3年）

### 【先进计算领域】项目指南

---

**项目名称**：基于网络拓扑可塑性的异构算力集群协同调度与跨任务迁移加速技术

**领域分类**：先进计算 / 高性能计算系统软件 / 智能计算系统

**核心问题**：
当前异构算力集群（GPU/TPU/NPU混合）的调度系统基于静态资源划分，各算力孤岛之间的互操作性差。跨任务迁移（如从大模型预训练迁移到下游推理）需要人工重配置，造成30-50%的资源空窗期。缺乏面向任务多样性的系统级动态资源调度理论。

**主要研究内容**：

**任务一：异构算力网络时空复杂度建模**
- 建立GPU/TPU/NPU混合集群的空间复杂度Sc量化方法（连通性/层级性/模块性）
- 研究训练/推理/微调三类任务流的时间动力学特征差异
- 建立跨集群互联网络（IB/NVLink/以太网混合）的Γst时空耦合量化方法
- 预期指标：CST建模框架覆盖≥3种异构互连拓扑，量化误差≤10%

**任务二：基于幂律流量检测的算力集群临界态感知**
- 研究算力集群激活传播的分支比（branching ratio）实时估计方法
- 建立集群从次临界→临界态→超临界的三态在线识别算法
- 研究临界态感知与任务调度决策的闭环反馈机制
- 预期指标：临界态识别延迟≤100ms，误判率≤5%

**任务三：类STDP的跨集群动态资源绑定机制**
- 研究任务共激活统计与资源路径权重增强的映射关系
- 建立"热路径固化"（LTP等价）与"冷路径修剪"（LTD等价）的资源回收策略
- 研究多集群之间的时序相关性驱动的资源预分配机制（预见性调度）
- 预期指标：跨任务资源切换开销相比现有方案降低≥50%

**任务四：系统级能效超线性增益的工程实现与验证**
- 在≥1000节点规模的异构集群上部署可塑调度系统
- 建立J/task、任务多样性D_task、迁移速度V_transfer三维度综合评测方法
- 与Kubernetes/Slurm等现有调度系统进行对比测试
- 预期指标：J/task降低≥25%，同等硬件并发任务类型数量提升≥2倍

**任务五：大规模集群间互联的拓扑自演化理论**
- 研究算力网络拓扑在任务驱动下的自组织演化规律（有向图可塑性）
- 建立骨架拓扑（E-L等价）的稳定性条件与可塑拓扑（E-S等价）的演化方程
- 探索多数据中心互联场景的跨域时空复杂度协同提升方法
- 预期指标：跨数据中心协同任务的端到端延迟降低≥20%

**预期重大成果**：
- 提出异构算力集群时空复杂度协同调度理论（1套）
- 开发可塑调度系统软件并完成千节点规模验证（1套）
- 发表SC/HPCA/ISCA/OSDI级论文≥8篇
- 申请发明专利≥10项，推动形成行业标准草案≥2项

**建议经费**：5000-8000万元（4年）

### 【微纳电子领域】项目指南

---

**项目名称**：面向算力网络可塑互连的软件定义晶上互连芯片关键技术研究

**领域分类**：微纳电子 / 先进封装与集成 / 智能互连芯片

**核心问题**：
现有算力互连芯片（交换芯片、SerDes、光引擎）固化了拓扑逻辑，无法在运行时根据任务动态重构互连路径。当前Chiplet封装和CoWoS技术实现了高密度集成，但互连拓扑仍为静态设计，缺乏物理层面支撑网络拓扑可塑性的器件基础。

**主要研究内容**：

**任务一：支持动态权重调控的可重构互连器件研究**
- 研究忆阻器/相变材料在互连权重动态调节中的应用（对应E-S/E-L化合键物理实现）
- 研究基于电可调液晶/MEMS的光互连路由动态重构技术
- 建立互连权重调节速度（<1ms）与功耗（<1pJ/bit）的综合优化方法
- 预期指标：权重调节速度≤500μs，调节功耗≤0.5pJ/bit，保持带宽损失≤5%

**任务二：晶上可塑互连网络的拓扑生成与演化规则硬件实现**
- 研究类STDP时序规则的轻量化硬件实现方案（专用逻辑单元，面积<0.1mm²）
- 研究短时程突触抑制（STD等价）的电路模型（RC充放电类比资源恢复动力学）
- 研究不应期机制的数字硬件实现（触发器+计数器，无需模拟电路）
- 预期指标：可塑规则硬件模块时钟频率≥1GHz，面积≤200μm×200μm

**任务三：多时间尺度互连动力学的片上监测与临界态感知**
- 研究片上流量幂律检测的轻量化算法及其ASIC实现（面积<0.5mm²）
- 建立分支比σ_branch的片上实时估计电路（基于简单计数器+移位寄存器）
- 研究临界态感知信号驱动的片上拓扑自动调节闭环电路
- 预期指标：片上幂律检测延迟≤10μs，功耗≤50mW

**任务四：软件定义互连（SDI）芯片原型设计与流片验证**
- 完成支持动态拓扑的互连控制芯片设计（≥28nm工艺）
- 集成可塑路由逻辑、STD等价电路、临界态感知模块
- 在≥64节点的Chiplet系统上完成功能验证
- 预期指标：芯片面积≤50mm²，功耗≤5W，支持≥256条动态可重构路径

**任务五：面向下一代算力网络的SDI标准化接口研究**
- 研究SDI化合键（E-L/I-L/E-S/I-S）在片上/片间/板间/机柜间的分层实现规范
- 建立与UCIe/BoW/AIB等现有Chiplet接口标准的兼容性方案
- 推动SDI接口标准化，参与IEEE/CXL联盟标准制定
- 预期指标：与UCIe标准兼容≥90%，提交标准草案≥1项

**预期重大成果**：
- 完成SDI互连控制芯片流片（1颗，≥28nm）
- 建立可塑互连物理实现规范（1套）
- 发表ISSCC/VLSI/DAC/DATE级论文≥6篇
- 申请核心发明专利≥12项

**建议经费**：8000万-1.2亿元（4年，含流片费用）
