# NaaS商业模式设计

## 产品线三层架构

```
Layer A：IP诊断服务（入口产品）
  → 最低门槛，建立信任
  → 定价：10-50万/次诊断

Layer B：IP核授权（技术产品）
  → 中等门槛，持续收入
  → 定价：200-1000万/次授权

Layer C：NaaS订阅（平台产品）
  → 最高价值，规模化收入
  → 定价：按节点数/月
```

## NaaS定价模型

| 集群规模 | 月费 | 年费（9折） | 包含服务 |
|---------|------|-----------|---------|
| 微型（≤8节点） | 20万 | 216万 | 互连芯粒+SDI软件+7×24运维 |
| 小型（8-32节点） | 80万 | 864万 | +定制拓扑优化 |
| 中型（32-128节点） | 300万 | 3240万 | +专属工程师驻场 |
| 大型（>128节点） | 定制 | — | 全栈服务 |

## 第一阶段目标（18个月）

```
收入目标：
  IP诊断服务：3-5个客户 × 30万 = 90-150万
  IP授权：1-2个授权 × 500万 = 500-1000万
  NaaS试点：1个客户 × 100万/年 = 100万
  合计：约700-1250万

团队规模：6-10人
  - 硬件工程师 × 2（FPGA+RTL）
  - 算法工程师 × 2（CST仿真+优化）
  - 客户工程师 × 2（诊断服务+集成）
  - 商务 × 1
```

## 核心竞争壁垒

```
1. 理论壁垒：CST RI指数是唯一可预测最优拓扑的量化工具
2. 专利壁垒：SDI化学键集合通信原语（申请PCT）
3. 数据壁垒：诊断服务积累的客户通信模式数据→持续优化RI模型
4. 关系壁垒：与中科院微电子所的联合流片通道（竞争者难以复制）
```
