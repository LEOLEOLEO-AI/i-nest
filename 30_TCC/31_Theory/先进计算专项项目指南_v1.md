# 先进计算重点专项 · 项目指南
> 方向：算力集群扩展效率 / 动态拓扑调控 / 超线性协同验证 / 标准化  
> 切入视角：当前信息系统与算力网络（第一类，1+1>2）  
> 版本：2026-03-23

---

## 核心论点

后摩尔时代算力扩展的核心矛盾已从单芯片性能转向系统级协同效率。当前大规模AI训练集群在扩展到256卡以上时，扩展效率普遍低于0.8——30%以上的算力被通信开销吞噬。

**性能评价范式的根本问题：** 现有"FLOPS/W"指标仅衡量计算节点能效，忽视了通信拓扑结构对系统整体效能的决定性作用。建立以"单位能耗系统有效吞吐量"为核心的新型评价体系，是后摩尔时代先进计算的关键基础。

## 项目一

### 立项名称
**《面向大规模语言模型训练的算力集群超线性扩展关键技术与示范》**

*英文：Key Technologies and Demonstration for Superlinear Scaling of Computing Clusters for Large Language Model Training*

### 问题切入
当前256卡LLM训练集群中：
- All-Reduce通信占端到端时间35%~50%
- 扩展效率（256卡vs理想线性）约0.75，1024卡约0.55
- 每增加一倍算力，仅带来0.6~0.8倍有效训练速度提升

**目标：** 通过互连拓扑优化+任务感知路由+动态重构三项技术叠加，将256卡扩展效率提升至≥0.90，实现算力投入与产出的超线性关系（1+1>2）。

### 研究内容
1. 研究面向LLM训练通信模式（All-Reduce主导）的专项拓扑优化方案，在256节点以上规模系统验证模块化层级互连拓扑相对Fat-Tree的协同增益
2. 研究任务感知路由与动态拓扑重构的协同优化，针对训练/推理/MoE混合负载自动切换最优拓扑
3. 建立算力集群拓扑优化的工程化方法论，形成可复制推广的设计规范
4. 在实际LLM训练任务（LLaMA-70B量级）上全面验证，建立标准化测试基准

### 关键指标
- 256卡集群扩展效率：≥0.90（相比理想线性扩展）
- LLM训练吞吐量（tokens/s）：相比Fat-Tree基线提升≥20%
- 系统总能耗：降低≥15%
- 混合负载（训练+推理）场景吞吐量：较固定拓扑提升≥40%

**申报渠道：** 科技部先进计算重点专项 / 工信部国产算力基础技术重大专项  
**经费建议：** 3000万—5000万 / 3年  
**合作落地：** 紫光算力、华为计算、阿里云、字节跳动

## 项目二

### 立项名称
**《AI算力网络临界性感知智能调度与动态优化系统》**

*英文：Criticality-Aware Intelligent Scheduling and Dynamic Optimization System for AI Computing Networks*

### 问题切入
当前AI算力集群调度（Kubernetes/Slurm等）不感知网络动力学状态，导致：
- 集群GPU利用率长期在30%~50%徘徊（过载与欠载交替）
- P99延迟波动大（突发任务响应慢）
- 扩容/缩容决策滞后（基于历史均值，而非实时状态预测）

统计物理的自组织临界性理论预测：系统工作在临界状态附近时，同时具备高吞吐量、低延迟方差和高突发弹性——**当前调度系统完全忽视这一物理原理**。

### 研究内容
1. 建立AI算力集群网络流量临界性的实时在线监测框架，基于流量统计特征计算系统动力学临界性指标
2. 研究临界性感知的任务准入控制：过临界前主动限流，亚临界时积极接入，使系统持续工作在临界区
3. 研究基于临界性预测的弹性扩容/缩容决策，实现前瞻性资源调度（预测窗口≥5分钟）
4. 开发与Kubernetes/Slurm等主流调度框架的集成插件，实现无感知部署

### 关键指标
- 稳态GPU利用率：从~40%提升至≥65%
- P99通信延迟：降低≥40%
- 突发任务响应时间：降低≥50%
- 弹性扩容决策提前量：≥5分钟（较响应式扩容）
- 支持集群规模：≥10,000节点

**申报渠道：** 科技部先进计算重点专项 / 工信部算力网络专项  
**经费建议：** 1500万—2500万 / 3年  
**合作落地：** 字节跳动（大规模算力调度）、百度/阿里云算力平台

## 项目三

### 立项名称
**《后摩尔时代算力体系性能指标范式转换与新型评价标准》**

*英文：Performance Metric Paradigm Shift and New Evaluation Standards for Post-Moore Computing Systems*

### 问题切入
现有"FLOPS/W"指标体系的根本缺陷：
1. 只衡量计算节点能效，不计入通信开销（实际系统中通信能耗占30%~50%）
2. 用峰值算力而非有效算力，掩盖了扩展效率损失
3. 无法区分"高算力低泛化"（如固化ASIC）与"中算力高泛化"（通用系统）

建立以"单位能耗系统有效吞吐量"和"拓扑复杂性—性能转化效率"为核心的新评价体系，是后摩尔时代先进计算布局的战略基础设施。

### 研究内容
1. 建立包含通信拓扑结构因子的算力体系综合能效评价指标，给出可操作化的测量标准
2. 系统分析现有主流AI芯片（GPU/NPU/存内计算/FPGA）在新指标体系下的上界，明确各类架构的结构性约束来源
3. 研究拓扑复杂性对系统有效吞吐量的贡献模型，建立"互连拓扑结构—算力效率"的定量关系
4. 推动相关指标纳入国家/行业标准体系

### 关键指标
- 新指标体系：完整定义，给出可操作化测量方法和归一化方法
- 覆盖≥10款主流AI计算平台的新指标实测对比
- 主导/参与国家/行业标准≥2项
- 国际标准提案（IEEE/IEC）：≥1项

**申报渠道：** 科技部先进计算重点专项 / 工信部标准化专项 / 国家标准化管理委员会  
**经费建议：** 800万—1500万 / 3年

## 项目四

### 立项名称
**《算力网络互连拓扑复杂性的超线性协同增益验证平台与标准体系》**

*英文：Validation Platform and Standard System for Superlinear Synergy Gain of Interconnect Topology Complexity in Computing Networks*

### 研究内容
1. **超线性协同验证：** 在统一测试平台上，量化拓扑复杂性优化、多时间尺度存储、动态重构互连三项技术叠加后的协同因子，验证整体增益超过各项技术独立增益之积（目标协同因子≥1.3）
2. **标准体系建立：**
   - 互连拓扑复杂性描述规范（拓扑结构特征标准化测量方法）
   - 复杂拓扑路由协议接口标准
   - 算力网络动力学状态监测数据格式标准
   - 算力网络拓扑复杂性评测基准（Benchmark）
3. **IP核体系商业化：** 开发可重用的复杂拓扑互连IP核，面向国产AI芯片适配验证

### 关键指标
- 三项技术叠加协同因子：>1.3
- 国家/行业标准：≥3项
- IEEE国际标准提案：≥1项
- IP核授权：向国内≥10家算力公司授权
- 专利：≥20项（发明专利≥15项）

**申报渠道：** 工信部算力标准化专项 / 国家标准化管理委员会  
**经费建议：** 2000万—3000万 / 3年  
**合作落地：** 紫光、海光、飞腾、寒武纪、天数AI（联合标准）
