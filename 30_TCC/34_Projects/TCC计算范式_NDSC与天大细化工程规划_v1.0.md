#### 1.3 NPC引擎微架构设计

##### NPC-FUSE (AllReduce) 微架构

`
                    Stage 1: Reduce          Stage 2: Broadcast
                   (Butterfly Network)       (Reverse Butterfly)

  Port 0 ──┬──────────────┐                   ┌──────────────┬── Port 0
           │   ┌──────┐   │                   │   ┌──────┐   │
  Port 1 ──┼───┤ ADD  ├───┼───────────────────┼───┤ DEMUX├───┼── Port 1
           │   └──────┘   │                   │   └──────┘   │
  Port 2 ──┼──────────────┼───────┐   ┌───────┼──────────────┼── Port 2
           │   ┌──────┐   │       │   │       │   ┌──────┐   │
  Port 3 ──┴───┤ ADD  ├───┴───────┴───┴───────┴───┤ DEMUX├───┴── Port 3
               └──────┘                           └──────┘

  Latency: log2(N) × (ALU_delay + wire_delay)
  N=4:  2 stages × 2ns = 4ns
  N=16: 4 stages × 2ns = 8ns
  N=64: 6 stages × 2ns = 12ns
`

| 子模块 | 功能 | 关键设计 |
|--------|------|---------|
| **ALU Array** | FP32/FP16/BF16/INT8归约运算 | 可配置精度，支持SUM/MIN/MAX/PROD |
| **Butterfly Network** | log2(N)级流水线互连 | 每级独立使能，可降级为Tree/Ring |
| **Result Buffer** | 归约结果暂存 | 双缓冲，支持流水线连续归约 |
| **Control FSM** | 操作模式控制 | 硬件状态机，无软件干预 |

#### 2.2 NPC原语消息格式定义

`
┌────────────────────────────────────────────────────────────┐
│                   SDIoN Frame Format                        │
├──────┬──────┬──────┬──────┬──────┬────────────────────────┤
│ Byte │ 0-1  │  2   │  3   │ 4-7  │       8-N              │
├──────┼──────┼──────┼──────┼──────┼────────────────────────┤

│                                                             │
│  Type Field:                                                │
│    0x01 = FUSE_REQ     (AllReduce 请求)                     │
│    0x02 = FUSE_RSP     (AllReduce 响应)                     │
│    0x03 = SWAP_REQ     (AlltoAll 请求)                      │
│    0x04 = SWAP_RSP     (AlltoAll 响应)                      │

│                                                             │
│  Flags Field:                                               │
│    Bit 0: 同步/异步标志                                      │
│    Bit 1: 最后分片标志                                       │

│                                                             │
│  Payload (FUSE_REQ 示例):                                   │
│    ┌──────┬──────┬──────┬──────────┐                        │
│    │Dtype │Rank  │Count │  Data[]  │                        │
│    │(FP32)│(0-63)│(bytes)│          │                        │

#### 5.2 CST理论→硬件参数映射

| CST参数 | 物理含义 | 硬件实现 | 可控范围 |

| S_c (空间复杂度) | 连接密度×聚类系数 | SDI端口数×互连拓扑 | 4-64端口/Chiplet |
| T_c (时间复杂度) | 同步精度×相干性 | 拓扑切换延迟×时钟精度 | 500ns-10μs切换 |
