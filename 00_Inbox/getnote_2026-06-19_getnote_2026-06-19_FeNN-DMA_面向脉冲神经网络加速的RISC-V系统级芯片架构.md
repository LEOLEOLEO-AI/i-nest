# FeNN-DMA：面向脉冲神经网络加速的RISC-V系统级芯片架构

### **🔬 研究背景与核心问题**

#### **(一) SNN的独特性与传统加速器的局限性**
- **脉冲神经网络(SNN)**：基于事件驱动机制，神经元仅在达到阈值时发放"脉冲"，具有类似大脑的稀疏通信特性，能效优势显著。
- **核心矛盾**：SNN计算的算术强度低、访存占比高，与GPU/TPU擅长的密集矩阵乘法不匹配，传统加速器难以高效支持。

#### **(二) 现有SNN加速器的三大限制**
1. **模型支持单一**：多限于简单LIF神经元模型。
2. **网络结构固化**：难以支持循环连接、突触延迟、非结构化稀疏。
3. **存储容量受限**：权重和神经元状态依赖片上BRAM，难以运行大规模SNN。

### **🎯 FeNN-DMA的设计目标与核心贡献**

#### **(一) 设计定位**

在灵活性、容量和能效之间取得平衡，构建可编程、可部署、支持复杂SNN的RISC-V FPGA平台。

#### **(二) 主要贡献**
1. **架构升级**：改进FeNN向量核心，加入lane-local memory支持稀疏连接和突触延迟。
2. **存储优化**：设计专用DMA控制器，高效在片外DDR和片上存储间搬运数据。
3. **工具链支持**：提供Python工具链PyFeNN，降低使用门槛。
4. **规模突破**：单核支持1.6万个神经元和2.56亿个突触。

### **🏗️ FeNN-DMA整体架构（图1解析）**

#### **(一) 三层架构设计**

| 层级 | 核心组件 | 功能描述 |
| :--- | :--- | :--- |
| **ARM处理系统** | ARM Cortex-A53 | 运行Ubuntu Linux和PyFeNN工具链，编译SNN模型为RISC-V代码 |
| **RISC-V标量核心** | CV32E40x | 执行控制逻辑（循环、分支、DMA调度） |
| **向量协处理器(VEC)** | 32-lane、512bit、3级流水线 | 并行处理SNN计算，支持自定义向量指令 |

#### **(二) 关键模块：DMA控制器**
- **组成**：AXI DataMover + Controller + Arbiter。
- **作用**：提前搬运片外DDR权重至片上URAM，隐藏访存延迟，实测带宽达4.9GB/s。

### **💾 存储系统设计：URAM、BRAM与Lane-local Memory**

| 存储类型 | 容量/特性 | 应用场景 |
| :--- | :--- | :--- |
| **BRAM** | 指令存储、标量数据 | 存放控制逻辑和标量变量 |
| **URAM** | 大容量片上向量存储 | 存放神经元状态、输入电流、临时缓冲 |
| **Lane-local Memory** | 每lane 18kbit BRAM（16bit宽） | 支持稀疏连接和突触延迟的独立寻址 |

**设计亮点**：URAM处理"大块连续数据"，lane-local BRAM处理"随机稀疏访问"，解决SNN数据访问模式多样性问题。

### **⚙️ SNN定制向量指令集（表1解析）**

#### **(一) 核心指令分类**
- **算术运算**：VADD（饱和加法）、VMUL（固定点乘法+右移）。
- **比较指令**：VTEQ（等于）、VTLT（小于）、VTGE（大于等于），用于脉冲发放判断。
- **访存指令**：VLOAD.V/VSTORE.V（向量存储访问）、VLOAD.L/VSTORE.L（lane-local memory访问）。
- **特殊功能**：VRNG（随机数生成）、VANDADD（掩码加法）。

#### **(二) 指令优化**
- **低精度支持**：针对SNN特性优化16bit固定点运算。
- **舍入模式**：支持round-to-zero、round-to-nearest和随机舍入，减少数值误差累积。

### **🔄 SNN执行流程与DMA双缓冲机制**

#### **(一) LIF神经元更新流程**
1. **状态打包**：32个神经元状态变量组成向量。
2. **向量运算**：用VADD/VMUL完成膜电位更新。
3. **脉冲判断**：用VTLT/VTGE比较阈值，脉冲压缩为bitfield。

#### **(二) DMA双缓冲技术（图3解析）**
- **工作原理**：两个URAM buffer交替使用，计算当前权重时DMA预取下一行权重。
- **效果**：隐藏DDR访存延迟，匹配脉冲处理循环的2时钟周期/512bit向量需求。

### **🔀 连接类型支持：稠密、稀疏与延迟**

#### **(一) 算法1实现的三种传播方式**

| 连接类型 | 实现方式 | 应用场景 |
| :--- | :--- | :--- |
| **稠密连接** | 直接加载权重矩阵行，向量加法更新输入电流 | 全连接层 |
| **稀疏连接** | 16bit编码（低位目标索引+高位权重），lane-local memory分布存储 | 高稀疏网络 |
| **延迟连接** | 多delay slot输入缓冲（图4B），脉冲写入未来时间槽 | 时序依赖网络 |

### **🐍 PyFeNN软件栈**
- **功能**：类似PyTorch的网络定义接口，C-like语言描述神经元更新规则，自动编译为RISC-V指令。
- **兼容性**：支持Neuromorphic Intermediate Representation，可导入其他SNN框架模型。

### **📊 实验结果与性能分析**

#### **(一) 硬件实现指标**

| 配置 | 频率 | 神经元支持 | 突触支持 | 动态功耗 | 稠密吞吐 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **单核** | 175MHz | 16,000个 | 2.56亿个 | 0.53W | 0.92 GSOP/s |
| **双核** | 143MHz | - | - | - | - |

#### **(二) 真实任务性能**

| 任务 | 模型特性 | 准确率 | 关键指标 |
| :--- | :--- | :--- | :--- |
| **SHD语音识别** | 带循环连接和可训练突触延迟 | 90.32±0.01% | 10.1ms/数字，双核加速40% |
| **N-MNIST图像识别** | 95-99%稀疏连接 | 98.46±0.02% | 参数量为稠密模型1/4 |
| **Braille触觉分类** | 量化模型 | 92.86% | 与最佳软件后端精度接近 |

#### **(三) 规模扩展能力（图6）**
- **实时仿真**：稠密连接支持10,000神经元，90%稀疏连接支持16,000神经元（快于实时）。
- **稀疏增益**：90%稀疏时有效吞吐提升3.2×，75%稀疏时提升2.1×。

### **💡 核心价值与创新点**
1. **瓶颈精准定位**：聚焦SNN的稀疏事件处理、权重搬运和存储协同问题。
2. **可编程性优先**：RISC-V+向量协处理器架构，支持复杂模型和网络结构。
3. **完整软件生态**：PyFeNN工具链降低使用门槛，促进科研落地。

### **⚠️ 局限与未来方向**

#### **(一) 当前局限**
- **吞吐差距**：相比固定功能加速器有3-4倍吞吐差距（脉冲传播循环依赖软件指令）。
- **多核协同**：双核仅支持数据并行，缺乏核间spike interconnect。

#### **(二) 未来计划**
1. **硬件加速**：将脉冲传播逻辑固化为流水线，提升小模型吞吐。
2. **多核扩展**：加入核间通信，支持大SNN拆分仿真。
3. **训练支持**：探索EventProp等事件驱动训练算法，扩展至更大FPGA平台。

### **🎯 总结：SNN硬件设计的务实路线**

FeNN-DMA展示了神经形态计算的中间路线：既避免GPU对密集计算的过度优化，又克服ASIC的灵活性不足，通过**RISC-V可编程性+向量并行+DMA存储优化+lane-local稀疏支持+Python工具链**的组合，为快速演进的SNN模型提供了高效、灵活的硬件平台。
