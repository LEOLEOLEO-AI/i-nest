## 一、杨立昆 JEPA 世界模型的最核心创新

### JEPA（Joint Embedding Predictive Architecture）的本质

JEPA 的核心创新不是"生成像素"，而是**在表征空间中做预测**：


| 传统生成模型（VAE/GAN/Diffusion） | JEPA           |
| ------------------------- | -------------- |
| 在原始像素空间中预测                | 在**联合嵌入空间**中预测 |
| 浪费算力重建无关细节                | 只预测**抽象表征**的演变 |
| 容易产生模糊/幻影                 | 天然避免像素级幻觉      |


**最核心的三点**：

1. **非生成式预测**：编码器 $E_x$ 和 $E_y$ 将输入和目标映射到潜在空间，预测器在潜在空间而非像素空间工作——这本质上是**在流形上进行预测**，天然与你的"结构流形吸引子"思想对齐
2. **能量模型作为约束**：通过对比能量函数（EBM）正则化，低能量=世界中的合理状态，高能量=不可能状态
3. **层级化 JEPA**：H-JEPA 堆叠多层预测器，对应世界模型的不同抽象层次

> **对你的直接启示**：JEPA 的潜在空间预测 + LNN 的连续时间动态 = 天然契合——LNN 恰好可以在 JEPA 的嵌入空间中以微分方程形式建模世界的连续演化。

### 2026 最新 JEPA 变体（ArXiv 刚出炉）：

- **Subspace-Decomposed JEPAs**（2026-05-29）：将潜在空间解耦为"进展"和"内容"子空间
- **UWM-JEPA**（2026-05-25）：在**信念空间**中做预测，而非原始感知空间
- **BRo-JEPA**（2026-05-31）：用潜在空间学习模运算，证明 JEPA 可以学到结构化推理

## 二、LNN 现状生态 + Transformer 映射 + 部署

### LNN 当前生态


| 层次      | 现状                                                                      |
| ------- | ----------------------------------------------------------------------- |
| **算法层** | CfC（Closed-form Continuous-time）是主力，有 ODE-RNN / LTC / CfC 三代演进          |
| **框架层** | 主要在 PyTorch 生态，Liquid AI 有内部 SDK；社区有 `ncps`（Neural Circuit Policies）开源库 |
| **硬件层** | 尚未有专用 LNN 芯片，但 LFM2 已在 CPU 上优化到比 Transformer 快 2x                       |
| **应用层** | 机器人、时序预测、边缘 AI、电池管理、医疗 —— 正从论文走向产品                                      |


### 与 Transformer 的映射关系

**不是直接映射，而是互补替代**：

```
Transformer:  Space → 离散 Token × 自注意力 → 全局交互
                     ↓
LNN:          Time  → 连续 ODE 状态演化 → 局部动态 + 因果性
```

- LNN **天然具备因果性**（单向时间演化），不需要 causal mask
- LNN 的 hidden state 是**连续时间轨迹**，不是离散的 token 序列
- 可以**混合使用**：LFM2 就是门控卷积（LNN风格）+ 分组查询注意力（Transformer风格）的混合体

### 应用部署路径

```
阶段1: PyTorch 训练 → ONNX 导出 → ONNX Runtime (CPU/GPU)
阶段2: 量化 (INT8) → 边缘部署 (TFLite/ONNX Micro)
阶段3: 硬件协同设计 → FPGA 加速 → 最终走向异步电路 ASIC
```

## 三、自由能原理驱动的自演化神经网络 —— 谁在做？

### 核心理论与人物


| 人物/机构                                      | 贡献                                |
| ------------------------------------------ | --------------------------------- |
| **Karl Friston**（UCL）                      | 自由能原理（FEP）创始人，Active Inference 框架 |
| **Wellcome Centre for Human Neuroimaging** | FEP 的理论中心和数学基础                    |
| **VERSES AI**                              | 将 Active Inference 商业化的公司，发布开源库   |
| **MIT CSAIL (Daniela Rus, Ramin Hasani)**  | LNN + 物理约束的连续时间网络                 |
| **Hessam Akhlaghpour** (Rockefeller)       | 神经调节作为动力系统，Neuro-Vesicles 模型      |


### 2025-2026 前沿论文

- **Neuro-Vesicles**（最新 ArXiv）：提出在神经网络中添加**动态囊泡群体**——每个囊泡有自己的浓度、扩散系数、寿命、目标受体，作为第二通信层，**不是张量修饰而是真实的动力系统**
- **EASE: Embodied Active Event Perception via Self-Supervised Energy Minimization**（2025-06）：通过自监督能量最小化实现主动感知
- **Bayesian Predictive Coding**（2025-03）：将预测编码与贝叶斯推断统一的框架
- **CogniSNN**：支持神经元可扩展性、路径可复用性、动态可配置性的随机图 SNN 架构

### 核心路线：从 FEP 到自演化收敛到吸引子

你的直觉非常准确。这条路上的关键数学链条：

```
变分自由能最小化
    ↓
最小惊讶度 (surprisal minimization)
    ↓
最小能量消耗 (least action principle)
    ↓
自发收敛到流形吸引子 (attractor manifold)
    ↓
结构自组织 (self-organized criticality)
```

**这正是 Friston 的 Active Inference 框架的核心数学**。不是人为设计激活函数，而是让网络通过**最小化自由能**自动发现最优动力学。

## 四、可直接借鉴的开源 IP 和代码

### 1. LNN / CfC 开源实现

- **ncps**（Neural Circuit Policies）：MIT 官方开源 `github.com/mlech26l/ncps`，含 LTC 和 CfC PyTorch 实现
- **CfC Pytorch**：`github.com/raminmh/CfC`，Closed-form Continuous-time 网络的参考实现

### 2. Active Inference / 自由能原理

- **pymdp**：Python 实现 Active Inference，含变分自由能计算 `github.com/infer-actively/pymdp`
- **SPM**（Statistical Parametric Mapping）：Friston 实验室的 MATLAB 工具箱，含 DCM（Dynamic Causal Modelling）

### 3. 预测编码

- **Predictive Coding Networks**：多个 PyTorch 实现，如 `github.com/RobertRosenbaum/PredictiveCoding`

### 4. SNN / 神经形态 / FPGA

- **snnTorch**：`github.com/jeshraghian/snntorch`，PyTorch 风格的 SNN 训练框架
- **Lava**：Intel Loihi 的软件框架 `github.com/lava-nc/lava`
- **SpiNNaker**：曼彻斯特大学的神经形态平台，开源工具链
- **Verilator / SpinalHDL**：可用于 FPGA 原型验证（异步电路仿真）

### 5. 神经演化

- **NEAT-python**：`github.com/CodeReclaimers/neat-python`，拓扑演化算法
- **PyGAD**：遗传算法驱动的神经网络架构搜索

## 五、可落地的推进路线图

基于你的愿景——**异步电路 + 脉冲驱动 + 软件定义互连 + 自由能自演化**——这里给出一个分阶段推进建议：

### Phase 0：仿真验证平台搭建（0-3 个月）

```
目标：在纯软件环境中验证核心假设
```


| 组件       | 技术方案               | 复用源    |
| -------- | ------------------ | ------ |
| 液态拓扑层    | PyTorch CfC / ncps | MIT 开源 |
| 变分自由能驱动  | pymdp + 自定义损失      | UCL 开源 |
| SNN 脉冲编码 | snnTorch           | 学术界开源  |
| 拓扑演化     | NEAT + 自定义算子       | 社区开源   |
| 软件定义互连仿真 | NetworkX + 动态图重写   | 自研     |


**关键验证问题**：

- 最小惊讶度能否驱动拓扑自主变化？
- 能量约束下能否自然收敛到稀疏脉冲模式？

### Phase 1：原型架构定义（3-6 个月）

```
目标：定义"液态拓扑化合件"的计算图 DSL
```

参考 LFM2 的混合骨干思路，定义你的 iNEST 单元：

```
iNEST Cell = {
    动态方程:      dL_h(t)/dt = f_θ(h(t), x(t), t)     ← 液态核心
    突触调制:      W_ij = g(φ_ij, ΔF_ij)                ← 自由能驱动
    脉冲门控:      s_i = Θ(v_i - v_th)                   ← SNN 激活
    拓扑化合:      G_t = rewrite(G_{t-1}, ΔF_t)          ← SDI 动态路由
    自演化:        θ_{t+1} = θ_t - η∇F(θ_t)             ← FEP 优化
}
```

### Phase 2：FPGA 加速验证（6-12 个月）


| 步骤  | 内容                                              |
| --- | ----------------------------------------------- |
| 2.1 | 用 HLS（Vivado HLS / Catapult）将核心 ODE 求解器映射到 FPGA |
| 2.2 | 实现异步 FIFO 作为脉冲传输通道（无需全局时钟）                      |
| 2.3 | 用 AXI-Stream 实现软件定义互连的硬件微架构                     |
| 2.4 | 对比 FPGA vs GPU 的能效比和延迟                          |


**可参考的 FPGA 开源项目**：

- `github.com/neuromorphicsystems` 相关 FPGA SNN 实现
- 2025 论文 *"Neuromorphic Processor Employing FPGA Technology with Universal Interconnections"* 直接相关

### Phase 3：异步电路 + 脉冲 ASIC 探索（12-24 个月）

```
目标：定义异步脉冲神经网络 ASIC 架构
```

- 采用 **bundled-data** 或 **delay-insensitive** 异步协议
- 每个 iNEST 单元作为独立异步模块，通过片上网络（NoC）互连
- 脉冲事件驱动计算 → 无活动时零功耗（物理能量最小化的硬件体现）
- 参考 Intel Loihi 2 和 IBM TrueNorth 的架构思想，但引入**自由能驱动的在线学习**

### Phase 4：自演化规模化（24 个月+）

```
目标：从原型到多尺度产品
```


| 规模    | 神经元数量 | 应用场景      |
| ----- | ----- | --------- |
| Nano  | ~10³  | 传感器端异常检测  |
| Micro | ~10⁴  | 个人设备自适应交互 |
| Meso  | ~10⁵  | 机器人实时控制   |
| Macro | ~10⁶  | 数据中心级世界模型 |

## 六、关键风险与建议

### 技术风险

1. **自由能计算开销**：变分推断在硬件中开销大，建议先用**局部自由能近似**（每个神经元仅基于邻居的惊讶度）
2. **异步电路验证困难**：建议先用 Verilator 做周期精确仿真
3. **自演化的稳定性**：需要引入**弹性保护机制**（like critical slowing down 指标监测相变）

### 可以立即开始的行动

1. **今天**：fork ncps + pymdp，搭建一个最小验证 pipeline（CfC 网络 + 自由能损失函数）
2. **本周**：跑通 LNN 在时序预测上的 benchmark，确认液态架构的性能基线
3. **本月**：将自演化机制作为拓扑搜索问题，用遗传算法 + FEP 在 MNIST/时序任务上验证
4. **下季度**：开始 FPGA 原型设计

### 建议的合作机构


| 机构                             | 对接方向                 |
| ------------------------------ | -------------------- |
| MIT CSAIL                      | LNN 理论基础与 CfC 改进     |
| Intel Neuromorphic Lab (Loihi) | SNN 芯片架构             |
| UCL (Friston 团队)               | 自由能原理的神经实现           |
| VERSES AI                      | Active Inference 工程化 |
| 曼彻斯特大学 (SpiNNaker)             | 大规模脉冲网络仿真            |
| 苏黎世联邦理工 (Giacomo Indiveri)     | 异步神经形态电路             |


---

这是一个可以持续 3-5 年的研究纲领，但好消息是**每个阶段都有现成开源代码可以直接复用和改造**，不需要从零开始。建议从 Phase 0 的最小验证 pipeline 起步，用 3 个月内跑通"LNN + 自由能损失 → 拓扑自演化"这条核心链路，拿到第一个有说服力的实验结果后再逐步铺开。
