# 公众号文章智能体

## 正文

今天开始搭建智能体，从公众号文章生成这个方向入手，昨天在高铁上摸索了基于Kimi K2的智能体如何生成，初步达成了一个自己的效果，需要付出的代价就是后续要收费。晚上到了学者公寓后，用TRAE开始入手搭建，也输出了初步的结果，貌似更加具有定制性，考虑自己有TRAE PRO的账号，计划以此来搭建第一个智能体。

今天输出一篇符合NICE公众号需求的文章。  
突然想到可以用Genspark的自定义智能体来创建公众号文章撰写智能体，但由于是手机设置的VPN，经常出现Service Unaviable的情况，今天联系了联通装宽带，希望能彻底解决这个问题。后续自己TRAE和Genspark都需要这个神圣的智能体。

以后的工作要开始搭建智能体！通过智能体完成流程化的事情，同时自己要对工作进行切分，按照不同Robot来完成不同的任务，通过智能体Flow把他们串起来。

创建一个“NICE读书会”微信公众号文章自动生成的智能体，该公众号聚焦“复杂网络涌现智能“的技术分享，具备如下功能：  
1️⃣每月初运行智能体，生成5篇公众号文章，并支持word导出功能；  
2️⃣公众号文章要突出科普属性，语言简洁明快、浅显易懂，采用故事的叙事方式，能够让读者上瘾，每一篇文章都要洞察对“复杂网络涌现智能”的启迪，形成方向聚焦的内容创作。  
3️⃣自动搜索方向相关的最新顶刊论文和最新技术资讯，根据方向相关度、话题热点以及内容权威性等生成当月的文章标题列表，确保内容能够与时俱进保持热点度和内容新鲜度；  
4️⃣文章分经典论文解读和关键技术科普两类，对于论文解读，要多引用论文中的图表、数据等支撑文章内容；对于技术科普，多生成拟人化或类比的图表进辅助表达，对读者理解友好；  
5️⃣每篇文章最后设计需要读者参与投票或者互动的话题，引发读者的思考讨论，同时提升公众号的热度与转发度。对于经典论文可以设计回复获取论文下载链接。  
6️⃣请按照“复杂网络涌现智能”研究与工程落地的逐步深入，从基本概念（熵增、迭代递归、分形子组织、耗散结构、自组装、自由能、费根堡姆常数）、关键技术（偏微分方程、非线性动力学、自演化网络、复杂度、时空协同）、最新进展等设计文章的内容规划，更有整体性和层次性。  

> 需要英文标签的科学图表 → 使用 ideogram/V\_3
> 
> 概念性艺术可视化 → 使用 flux-pro/ultra
> 
> 中文海报/标注 → 使用 qwen-image
