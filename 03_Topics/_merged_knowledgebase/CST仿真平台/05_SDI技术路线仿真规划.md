### SDI拓扑生成策略（基于化合键+元拓扑）

```python
# SDI拓扑生成伪代码框架
def generate_sdi_topology(n_chiplets, target_sigma=5.0, k_ports=16):
    """
    使用化合键（Valence Bond）规则生成类小世界拓扑
    1. 先建立基础环形结构（保证高聚类）
    2. 按化合键规则添加跨模块连接（引入小世界性质）
    3. 迭代优化直到σ达到目标值
    """
    # 基础层：同一功能模块内高密度连接
    G = build_functional_clusters(n_chiplets, cluster_size=8)
    
    # 化合键层：按元拓扑规则添加模块间连接
    G = add_valence_bonds(G, bond_rules=META_TOPO_RULES)
    
    # 验证小世界系数
    sigma = compute_sigma(G)
    assert sigma >= target_sigma, f"σ={sigma:.2f} < target={target_sigma}"
    
    return G
```

## 阶段3详细设计：功能仿真

### 实验3.1：信息传播效率 vs 拓扑复杂度
- **方法**：扫描σ从1.0到8.0，测量全局效率、传播延迟
- **预期结论**：σ超过某个阈值（~4.0）后，效率出现超线性提升
- **指标**：全局效率E_glob，单位：1/平均最短路径

### 实验3.2：并行任务处理能力 vs 模块化程度
- **方法**：向不同功能模块注入并行信号，测量分离度
- **关键改进**（相比线虫仿真）：加入功能标注（AI推理/存储/通信模块）
- **预期结论**：聚类系数C>0.3的SDI拓扑能并行处理多路任务流

### 实验3.3：容错能力 vs 网络冗余度
- **方法**：随机故障+针对性攻击两种场景
- **目标**：找到σ与容错能力的最优权衡点
- **工程意义**：确定SDI设计的最小冗余互连比例

### 实验3.4：规模涌现阈值
- **方法**：从N=16扫描到N=1024芯粒，找到功能质变阈值
- **参照**：C. elegans N=302（第二级智能阈值θ₂=1.0）
- **预期**：找到SDI系统的工程实现"最小可用规模"
