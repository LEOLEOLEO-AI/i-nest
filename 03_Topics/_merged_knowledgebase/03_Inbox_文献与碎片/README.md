# 集合通信硬件化 + NaaS产业落地项目

> 基于CST时空协同复杂度理论 × SDI化学键机制  
> 在晶圆级网络上硬件化实现 Allreduce / Alltoall  
> 从IP诊断服务切入，演进为集合通信NaaS平台

---

## 项目定位（一句话）

**不卖芯片，卖"网络智能加速能力"——让每一块AI芯片跑得更快、更省、更自主。**

---

## 核心创新点

### 与国际方案的差异化
| 方案 | 层级 | 局限 |
|------|------|------|
| NVIDIA SHARP | InfiniBand交换机内in-network reduction | 只做Allreduce；依赖IB生态；软件栈重 |
| Cerebras WSE-3 | 全晶圆NoC | 单芯片内部；拓扑不可重构；专有协议 |
| Enfabrica ACF-S | GPU外挂NIC | 外挂架构；不是晶圆级集成 |
| **本方案（SDI+CST）** | **晶圆级可重构互连+硬件集合通信原语** | — |

### 三个叠加差异化
1. **做进晶圆**：可重构晶圆级网络（vs Cerebras固定拓扑）
2. **化学键机制**：SDI连接方式本身是可编程的物理原语
3. **CST理论支撑**：RI=CST/E_env，可预测最优拓扑（vs 经验调参）

## 发展路径

```
阶段1（0-6月）：IP诊断服务
  → 给AI芯片厂商做"集合通信性能诊断"
  → 输出：瓶颈报告 + 优化建议
  → 收入：顾问费/技术服务费
  → 目标客户：寒武纪、燧原科技、壁仞科技

阶段2（6-12月）：IP核授权
  → 硬Allreduce/Alltoall IP核授权给厂商集成
  → 收入：IP授权费（ARM模式）
  → 交付：FPGA原型验证 + RTL设计包

阶段3（12-18月）：NaaS平台
  → "集合通信加速即服务"
  → 厂商提供算力芯片，我方提供互连芯粒+SDI软件
  → 收入：按集群规模/吞吐量收费
  → 目标：第一个付费客户
```

## 关键指标目标（MVP验证标准）

| 阶段 | 指标 | 目标值 | 对标 |
|------|------|--------|------|
| FPGA原型 | Allreduce延迟 | <2μs | SHARP ~10μs |
| 小系统集成 | Allreduce吞吐 | ≥2TB/s | NVLink ~600GB/s |
| 功耗 | 4芯粒集成 | <20W | GPU同等规模 ~300W |
| 延迟方差 | 抖动 | <5% | 软件方案 30-100% |
