# 史上最全类脑软件总结。大脑仿真也好奇：什么时候大模型可以趋近人脑的数量级？

> 笔记本: 我的剪贴板  
> 创建时间: 2024-03-22  

---

**前言：**类脑计算 (Neuromorphic Computing) / 脉冲神经网络 (Spiking Neural Network, SNN)是在人工智能领域里离神经科学最近的学科，甚至可以定义，在某种程度上，脉冲神经网络是人工智能和计算神经科学的交叉点。人类大脑或许是我们所知的宇宙中最复杂最智能的存在，约1000亿个神经元，100-1000万亿个突触。全球层出不穷的脑计划 (美国BRAIN Initiative，中国China Brain Project，欧洲Human Brain Project，瑞士Blue Brain Project，日本Brain/MINDS等) 投入巨大的财力、人力、智力，试图窥见那一点智慧的火花 。 
据传GPT-4的参数量级大概在1万亿这个级别，距离人脑的突触量还有2-3个数量级，我相信，人类总有梦想解构智能与智慧，总有梦想推开门看一眼，总有关于一切的好奇心。 
Neuromorphic Computing就是源于对大脑强烈的好奇心，由此驱使而诞生的学科，在中文语境下翻译成“类脑”，直译为“神经形态计算”。 
本质上来说，计算是一种物理过程，而非纯粹的数学过程。那么，如何才能在硬件上捕捉和复现大脑中的计算过程？摩尔定律的提出者Carver Mead(美国计算机和电子电气科学家)在1989年给出了他的思考：“基于硅的系统面临的限制与神经系统是类似的：连接是有限的，功耗是宝贵的，鲁棒性和可靠性是必不可少的。” 
或许Carver Mead看到了摩尔定律的最终限制，从而专注于大脑的启发，与Terry Sejnowski (美国五院院士，计算机与神经科学家)，Rodney Douglas (瑞士计算神经科学家)，Christof Koch (美国神经生理学家和计算神经科学家)一起开始探讨研究，每年夏天在美国科罗拉多州的一个美丽小镇Telluride举办研讨会，整日一起头脑风暴爬山踏青，这便是类脑计算的开始。 
如果我们浅浅将类脑计算分成软件工作和硬件工作，那么，尝试模拟仿真人脑的软件，必然是极其重要的存在，毕竟，不仿真怎么做芯片！没有框架怎么研究大脑！于是，就有了百花齐放的类脑框架。 
我们先将“**类脑软件**”分为“**数据处理框架**”和“**算法框架**”。 
为什么需要数据处理框架？因为我们类脑计算领域的传感器也很特别，也是大脑启发的传感器，比如事件相机 (Event Camera, Dynamic Vision Sensor)，不同于传统按帧捕捉图像的相机，事件相机内的每个像素都独立且异步地运作，当亮度变化超过阈值的时候发出脉冲，其他时候则没有输出，这类似人眼的注意力机制。 
在算法框架中，有更贴近计算神经科学的框架，也有更贴近深度学习的框架，还有尝试两者兼容的框架。 
不仅如此，还有框架之上的框架！ 
比如最近的工作Neuromorphic Intermediate Representation (NIR)，它尝试回归动力学的基础计算单元，在所有算法框架和硬件芯片中搭建一个物理运算的桥梁。总之是眼花缭乱，充分说明这个领域还是充满着创造力与混沌力 :) 
笔者搜集与总结了以下二十多个类脑相关软件，排名不分先后。如果还有未覆盖到的，以及任何建议，或者是对开源类脑感兴趣的朋友，请后台联系！ 
封面是笔者认为去年最好玩的类脑计算梗图，来自论文“Richards, Blake Aaron, and Konrad Paul Kording. "The study of plasticity has always been about gradients." The Journal of Physiology 601.15 (2023): 3141-3149.” 
  
论文作者Blake Aaron Richards是来自加拿大的神经科学家，Konrad Paul Kording是来自美国的神经科学家，他们尝试从神经科学的角度理解人工智能领域的梯度下降学习算法。长久以来神经科学领域都在探索人脑的学习机制，特别看重人脑的“无监督学习”、更偏“本地”而非梯度下降这样的“全局”学习机制，如Spike-timing dependent plasticity (STDP)，但是让神经科学家也很惊讶的是，梯度下降算法竟然如此简单有效，所以现在学术界变成了一种“双向奔赴”。 
我们尝试将大脑的学习机制应用在人工智能应用中，也同时尝试在大脑中寻找梯度下降算法的证据。Konrad Paul Kording就是非常具有代表的持有“大脑中一定有梯度下降算法”这一认识的神经科学家，论文中一头蓝色短发闪亮耀眼。于是： 
**“Wait, it's all just gradients? Always has been.”**

 **1.1 Tonic**  Tonic是一个用于管理和转换类脑数据集的Python包。 网站：https://tonic.readthedocs.io/ 源代码：https://github.com/neuromorphs/tonic 许可证：GPL-3.0  Tonic 是一个专门的 Python 包，旨在便于下载和操作类脑数据集，特别是专注于基于事件的视觉和音频数据。它与 PyTorch Vision/Audio 完全兼容，并提供了一系列事件转换，使其成为处理类脑数据的灵活工具。该包包括各种公开可用的数据集，并为这些数据集的管理和转换提供了高效的方法，以适应不同的应用。  Tonic 的文档包括有关加载和增强图像以及事件的教程、缓存数据以加快加载速度、事件数据的批处理以及数据集的切片以便于管理处理。此外，它还提供了其类脑数据集和事件转换的 API 参考，并支持各种格式的文件解析器。对于那些新入门该领域或希望加深理解的人，Tonic 提供了关于类脑相机、脉冲神经网络及相关模拟器的阅读材料。  Tonic 是一个开源项目，鼓励社区参与，提供了贡献指南和协作的沟通渠道。该项目在其发布说明中概述了其发展历程和更新情况，提供了关于其演变和背后社区的洞察。 
 **1.2 AEStream**  AEStream是一个高效传输事件数据的工具，支持多种输入/输出，并与Python和C++库集成。 网站：https://aestream.github.io/aestream 源代码：https://github.com/aestream/aestream  许可证：MIT  AEStream 是一个先进且灵活的工具，专为高效处理和传输基于事件的数据而设计，满足类脑计算和基于事件的传感的独特需求。它能够与各种数据源接口，包括不同型号的事件相机、网络流和数据文件，使其高度适应各种应用。AEStream 支持多种输入和输出格式，并可在不同环境中使用：作为命令行工具、通过 Python 接口或作为 C++ 库，允许用户选择最适合其工作流程的方法。  该工具不仅能够处理静态数据，而且在实时流媒体场景中表现出色，例如直接与事件相机的 USB 或 UDP 输入接口。这使得 AEStream 对于需要实时数据处理的应用程序特别有价值，如机器人动态视觉系统或研究环境中的实时数据分析。其对广泛的事件相机模型和文件格式的支持确保了兼容性和易于集成到现有系统中。  AEStream 的一个核心优势是其能够与流行的 Python 库 (如 Numpy 和 PyTorch)集成，允许用户在 Python 数据科学和机器学习工具的更广泛生态系统内轻松处理和操作事件数据。文档提供了关于使用 AEStream 进行实时应用的示例和指南，如使用脉冲神经网络进行边缘检测，展示了该工具对于高效和复杂的事件数据处理的能力。  作为一个开源项目，AEStream 鼓励社区参与和贡献，旨在持续改进和扩展其功能。它既考虑到易用性又注重高性能，使其成为基于事件的数据领域中研究人员和从业者的合适选择。 
  **1.3 AEDAT**  AEDAT是一个Python包，用于解码来自事件相机的AEDAT 4文件，采用Rust实现以提高速度。 网站：https://pypi.org/project/aedat/ 源代码：https://github.com/neuromorphicsystems/aedat 许可证：MIT  项目 aedat 是一个位于 GitHub 上的快速 AEDAT 4 Python 解码器，采用 Rust 实现，允许用户高效地读取 .aedat 数据文件。它便于处理类脑计算和视觉系统中常用的基于事件的数据。用户可以轻松地使用 pip 安装该库，并利用流行的 Python 库如 Pillow 和 OpenCV 来读取和处理帧。该存储库包括关于创建解码器对象、遍历数据包以及处理不同类型的事件或帧的示例和详细说明。aedat 采用 MIT 许可证授权，是一个旨在处理 AEDAT 文件时提供灵活性和速度的开源工具。  
 **1.4 Expelliarmus**  Expelliarmus能将事件相机数据解码成NumPy数组，支持多种格式，为研究人员和开发人员提供易用性。 网站：https://expelliarmus.readthedocs.io/ 源代码：https://github.com/open-neuromorphic/expelliarmus 许可证：GPL-2.0  Expelliarmus 是一个专门的 Python 库，专注于解码来自基于事件的传感器的二进制数据，特别设计用于处理事件相机中流行的各种二进制格式，如 DAT、EVT2 和 EVT3。它将这些二进制数据转换为 NumPy 结构化数组，使得在 Python 生态系统内的操作和分析变得更加容易。这使得它成为类脑计算、机器人技术和计算机视觉领域的研究人员和开发者的重要工具，这些领域的专业人士依靠事件相机捕获以事件形式而非传统帧的视觉信息。  该库设计为跨平台，可在 Windows、MacOS 和 Linux 上无缝运行，确保了广泛的可访问性和集成到各种工作流程中。其用户友好性通过 pip 上的可用性进一步放大，允许简单安装和快速设置。文档提供了基准测试来展示其性能和效率，以及一个简洁的“入门”指南，帮助新用户快速将该工具应用于他们特定的需求。  Expelliarmus 支持从类脑工程和计算机视觉的学术研究到机器人技术等领域的实际应用，以及其他需要实时、高效处理动态视觉信息的领域。通过简化解码和处理事件数据的过程，Expelliarmus 有助于更广泛地采用和应用基于事件的感测技术，促进在各种技术领域中更高效、更有效地处理视觉信息。 **2. 算法框架**  **2.1 基于深度学习的SNN框架** 
 **2.1.1 snnTorch**  专注于基于梯度的SNN训练，基于PyTorch，支持GPU加速和梯度计算。 网站：https://snntorch.readthedocs.io 源代码：https://github.com/jeshraghian/snntorch 许可证：MIT  snnTorch是一个为脉冲神经网络模拟设计的Python包，它扩展了PyTorch对脉冲神经元的支持，提供了PyTorch框架内的预设计模型。它旨在弥合大脑高效的基于脉冲的信息编码和现代深度学习技术之间的差距。snnTorch包括了深度集成自动微分 (autograd）的脉冲神经元库、用于交叉兼容性的导出功能、对脉冲的算术操作、脉冲生成和数据转换的库、可视化工具、替代梯度函数以及数据集的实用函数。  该框架允许将脉冲神经元无缝集成到PyTorch的计算图中，将它们视为循环单元。它支持由递归函数表示的各种神经元模型，消除了存储所有神经元的膜电位迹的需要，这有助于在CPU和GPU上训练小型和大型网络。只要网络模型和张量被加载到CUDA上，用户就可以像在PyTorch中一样受益于GPU加速。  snnTorch由加州大学圣克鲁斯分校的类脑计算小组持续维护，并邀请社区贡献。其源代码在MIT许可证下发布，其文档在创作共用署名-相同方式共享3.0未本地化许可证下授权。该项目强调易用性、效率和可扩展性，使其成为对探索和推进脉冲神经网络领域感兴趣的研究人员和从业者的宝贵工具。 
 **2.1.2 SpikingJelly**  基于PyTorch的开源DL框架，用于SNN，提供英文和中文文档。 网站：https://spikingjelly.readthedocs.io 源代码：https://github.com/fangwei123456/spikingjelly 许可证：未知  SpikingJelly 是一个基于 PyTorch 的开源深度学习框架，专为脉冲神经网络 (SNNs)设计。它利用了 PyTorch 的用户友好界面和计算能力，包括 GPU 加速。这种整合使 SpikingJelly 成为神经科学和人工智能领域的研究人员和从业者的一个易于访问且强大的工具。  SpikingJelly 的目标是通过结合神经动力学和脉冲特性，为具有高能效的类脑芯片带来灵感来源于大脑的智能。SpikingJelly 的独特之处在于它满足了脉冲深度学习中的自动微分、并行计算加速和类脑数据集整合的需求。该框架为预处理类脑数据集、构建深层SNNs、优化参数和在类脑芯片上部署提供了一整套工具。  由北京大学数字媒体研究所多媒体学习小组和鹏城实验室开发的 SpikingJelly，正在促进能效高、基于脉冲的机器智能系统的发展。它是推进 SNN 研究的宝贵工具，为探索脉冲神经网络的动态世界提供了易用性和强大的计算工具的完美结合。 
 **2.1.3 Rockpool**  用于SNN应用的机器学习库，支持GPU、TPU、CPU加速和类脑计算硬件部署。 网站：https://rockpool.ai 源代码：https://gitlab.com/synsense/rockpool 许可证：AGPL-3.0  Rockpool 是一个专注于动态神经网络架构的开源 Python 包，专为事件驱动网络和类脑硬件量身定制。由 SynSense 管理，Rockpool 促进了具有连续时间或事件驱动动态的递归神经网络的设计、训练和评估。该库的设计注重效率，能够快速模拟和训练网络，这对于实时应用和在低功耗类脑硬件上的部署至关重要。  该框架提供了标准模块、用于处理时间序列数据的工具，以及针对 Jax 和 Torch 网络的专门训练技术。它提供了广泛的 API 并支持各种训练方法，包括梯度下降和对抗性训练。此外，Rockpool 能够与特定类型的硬件接口，如 Xylo™ 推理处理器、Xylo™ Audio、Xylo™ IMU 和 DYNAP-SE2 混合信号处理器，提供用于快速启动和训练针对这些设备的网络的资源。  Rockpool 以其用户友好的界面和与 Python 的集成而脱颖而出，使其对从研究人员到 AI 和神经科学领域的从业者的广泛用户都易于访问。此外，它还提供了用于分析和可视化神经数据的工具，帮助理解复杂网络行为。Rockpool 的文档包括教程，并涵盖了高级主题，如计算图和图映射、参数处理、性能基准测试和全面的 API 摘要。该项目还提供了开发者文档，包括 UML 图和后端管理的说明。旨在简化和优化在各种硬件平台上设计和部署神经网络的过程，弥合动态神经建模和实际应用之间的差距。 
 **2.1.4 Sinabs** 基于PyTorch的深度学习库，用于脉冲神经网络(SNNs)，专注于简单性、快速训练、可扩展性和视觉模型。 网站：https://sinabs.ai 源代码：https://github.com/synsense/sinabs 许可证：AGPL-3.0  Sinabs (Sinabs Is Not A Brain Simulator)是一个基于PyTorch的深度学习库，专门为脉冲神经网络设计。它专注于简单性、快速训练和可扩展性，在支持权重转移的情况下，特别是在视觉模型方面展示了强大的性能。该库为用户提供了一种高效的方式，将现有的人工神经网络转换为脉冲神经网络。其教程涵盖了各种流程，如将现有的ANN转换，或使用类脑版本的数据集 (如MNIST)运行使用时间反向传播 (BPTT)的示例。 Sinabs还配备了插件来增强其功能，包括将模型部署到类脑硬件上以及显著加速前馈模型的训练。API参考提供了对支持的神经元模型、权重转移API和其他功能的全面概述。文档提供了关于如何开始使用Sinabs的洞察，无论你是在探索脉冲神经网络的语法，还是希望将现有的神经网络适配为脉冲网络。特别值得注意的是，Sinabs与其他工具（如Rockpool）合作，满足不同类型的数据或后端需求。Sinabs鼓励社区参与，提供了明确的贡献指南，由SynSense维护。 
 **2.1.5 BindsNET**  基于PyTorch构建，用于模拟SNNs，针对ML和强化学习。 网站：https://bindsnet-docs.readthedocs.io/ 源代码：https://github.com/bindsnet/bindsnet 许可证：AGPL-3.0  BindsNET是一个开源的计算框架，旨在模拟脉冲神经网络(SNNs)。它建立在PyTorch深度学习库之上，由Hazan Hananel和Daniel Saunders在2018年创建。他们的工作得到了他们获得的国防高级研究计划局 (Defense Advanced Research Project Agency)的资助。BindsNET提供了创建、管理和模拟具有突触的脉冲神经元网络的工具和功能。它利用了PyTorch的GPU/CPU加速能力，充分发挥了SNNs的低功耗特性。此框架还附带有广泛的文档，包括安装指南、用户手册和详细的参考资料，使其对计算神经科学和机器学习领域的研究人员和实践者来说易于接近。  该框架支持多种不同类型的神经元模型和学习算法。它提供了多样性，允许特定的神经元模型之间的连接，以及不同类型的突触强度和连接。这种灵活性对于设计自己的网络架构的实践者和研究人员来说是非常宝贵的。BindsNET允许自定义神经元模型，使用户能够修改权重、张量偏置、权重的最大值以及所有权重的标准化因子，这对于突触可塑性、学习和记忆至关重要。在创建网络时，你可以指定一个模拟时间步长常数dt，这决定了模拟的粒度。时间步长参数引入了模拟速度和数值精度之间的权衡：较大的值结果在模拟速度更快，但准确性降低。  虽然BindsNET为SNN研究和应用打开了许多可能性，但它可能需要熟悉PyTorch和对SNN原理的扎实理解。然而，这一要求并不减少BindsNET库的多功能性、可定制性和实际应用性。

 **2.3.3 CARLsim**  是一个利用GPU加速的库，用于模拟具有高度生物学真实性的突触动态的大规模脉冲神经网络 (SNN)模型。 网站：https://uci-carl.github.io/CARLsim3/ 源代码：https://github.com/UCI-CARL/CARLsim6 许可证：MIT  CARLsim 是一个高效、易用、GPU加速的库，用于模拟具有高度生物细节的大规模脉冲神经网络 (SNN)模型。CARLsim 允许在通用 x86 CPU 和标准现成 GPU 上执行具有现实突触动态的Izhikevich脉冲神经元网络。模拟器提供了一个类似于 PyNN 的 C/C++ 编程接口，允许在突触、神经元和网络级别指定细节和参数。  一些特点包括：支持 CUDA 11、使用 CMake 构建系统、神经调节特性、集成 Python LEAP、轴突可塑性学习规则 (版本 6.1)、一个更完整的 STDP 实现，包括神经调节机制、一个自动化的参数调整界面，利用进化算法构建功能性 SNNs、用于功能性代码验证的测试套件。CARLsim 为研究人员和从业者提供了一个强大的工具，以模拟和研究脉冲神经网络，特别是在需要利用 GPU 加速来处理大规模网络的场景中。通过其详细的编程接口，用户可以精细地控制网络的每个方面，从而探索复杂的神经动力学和学习规则。  **2.4 计算神经科学通用表达框架**
 
  **2.4.1 PyNN**  PyNN是一种与模拟器无关的语言，用于构建神经网络模型。 网站：https://neuralensemble.org/PyNN/ 源代码：https://github.com/NeuralEnsemble/PyNN/ 许可证：自定义  可以使用PyNN API和Python编程语言编写一次模型代码，然后在PyNN支持的任何模拟器上无需修改即可运行 (目前支持NEURON、NEST和Brian)，以及在SpiNNaker和BrainScaleS类脑硬件系统上。  PyNN API旨在支持高层次的抽象建模 (神经元群体、层、列及其之间的连接)，同时在需要时仍允许访问单个神经元和突触的细节。PyNN提供了一个标准神经元、突触和突触可塑性模型的库，这些模型已经验证在不同支持的模拟器上工作相同。PyNN还提供了一组常用的连接算法 (例如全连接、随机、距离依赖、小世界)，但它使得在与模拟器无关的方式上提供自己的连接变得简单，可以使用连接集代数 (Djurfeldt, 2010)或编写自己的Python代码。低级API适用于小型网络，并可能提供更多的灵活性。高级API适用于隐藏细节和记录保持，让您可以专注于模型的整体结构。  在多个模拟器上编写一次模型并运行它所需的另一件事是标准细胞模型。PyNN将标准细胞模型名称和参数名称转换为特定于模拟器的名称，例如标准模型IF_curr_alpha在NEST中是iaf_neuron，在NEURON中是StandardIF，而SpikeSourcePoisson在NEST中是poisson_generator，在NEURON中是NetStim。即使您不希望在多个模拟器上运行模拟，使用PyNN强大的高级接口编写模拟代码也可能会有益。在这种情况下，您可以使用模拟器支持的任何神经元或突触模型，并且不受限于标准模型。  
 
**2.4.2 NeuroML** NeuroML是用于计算神经科学的模型描述语言。  网站：https://docs.neuroml.org/ 源代码：https://github.com/NeuroML 许可证：LGPL-3.0 license  基于详细的神经解剖和电生理数据的计算模型被广泛用作理解神经系统的辅助工具。NeuroML是一个国际合作倡议，旨在开发一种描述神经系统详细模型的语言，该语言将作为定义和交换神经元细胞和网络模型描述的标准数据格式。NeuroML规范由NeuroML编辑委员会开发，并由其科学委员会监督。NeuroML得到了国际神经信息学计算基础设施组织 (INCF)的认可，并且是官方COMBINE标准。NeuroML项目社区开发了一种基于XML (可扩展标记语言)的描述语言，其中使用XML模式定义模型规范。社区还开发和维护了多个库 (在Python、Java和其他语言中)以便于使用这些规范。  NeuroML倡议的目标包括：创建一个基于XML的语言的规范，描述神经系统的生物物理学、解剖学和网络架构，涵盖多个尺度。促进研究人员之间复杂神经模型的交换，提高模型的透明度和可访问性。推广支持NeuroML的软件工具，并支持新软件和神经建模数据库的开发。鼓励研究人员将其模型转换为NeuroML格式，并在该格式下交换和发布他们的模型。  
**2.5 类脑通用表达框架** 
 **2.5.1 NIR**  NIR (Neuromorphic Intermediate Representation)是一组计算原语，跨不同的类脑框架和技术堆栈共享。 网站：https://nnir.readthedocs.io/en/latest/ 源代码：https://github.com/neuromorphs/NIR 许可证：BSD-3-Clause license  NIR为类脑系统中的计算建立了一个共同的参考框架，称为类脑中间表示 (NIR)。NIR定义了一组作为理想化连续时间混合系统的计算原语，这些原语可以组成图并映射到各种类脑技术堆栈。通过抽象化围绕离散化和硬件约束的假设，NIR捕捉了基本计算，同时暴露出评估实现与理想化数学形式之间的确切差异。目前，NIR得到了7个类脑算法框架 (Lava-DL, Nengo, Norse, Rockpool, Sinabs, snnTorch, Spyx)和4个类脑硬件平台 (SpiNNaker2, Xylo, Speck, Loihi)的支持，允许用户在这些平台之间无缝切换。NIR，解耦了类脑硬件和软件的发展，最终增加了平台间的互操作性并提高了对类脑技术的可访问性。NIR是朝着继续研究脑启发硬件和旨在深化对神经系统计算基础的理解的自下而上方法的重要一步。    **** **总结：**乱花渐欲迷人眼！如笔者所说，这二十来个框架充分说明这个领域充满着创造力与混沌力。框架千千万，用顺手就行，不管如何，非常感谢全世界有如此多愿意做开源类脑框架的研究者，持续贡献着自己的代码量和发量 :)   **Acknowledgement****：**感谢“脑界漫游指南”的罗博士、类脑领域的同行师博士、类脑计算的爱好者Henry同学给予笔者写作建议。
