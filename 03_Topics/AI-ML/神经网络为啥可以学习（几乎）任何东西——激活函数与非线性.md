**神经网络中的线性**
我们已经解答了为什么在神经网络中需要非线性，但让我们更深入地探讨这个概念及其对模型的意义。

基于这个概念，现在考虑一个具有两层隐藏层的前馈多层感知机（MLP），其中中间层有两个神经元（忽略偏置项）：

其实，我们在这里所做的就是将两层MLP压缩成单层MLP！
在上述推导中的最终方程，实际上就是一个具有特征x_1和x_2及其对应系数的线性回归模型。
因此，尽管MLP是“两层”的，但它会退化成单层，并变成传统的线性回归模型！
这并不好，因为神经网络将无法对数据建模或拟合复杂函数。
通过允许MLP使用非线性激活函数，我们可以满足所谓的通用逼近定理。
该定理基本上表明，MLP（或更确切地说，神经网络）可以逼近和拟合任何函数。
线性并不一定是坏事，只是许多实际问题都是非线性的，因此如果我们想预测这些现象，就需要使用非线性模型。
**激活函数**
现在，让我们快速了解一下一些最常见的非线性激活函数。
Sigmoid
Sigmoid函数在业界广为人知，它源自二项分布。
它将输入“挤压”到0和1之间的输出，因此可以将其视为概率，并具有“S形”曲线。
然而，最重要的是，它是非线性的，因为它包含除法和指数函数。
Sigmoid函数的数学和视觉表示如下：


# Import packages
import numpy as np
import plotly.express as px

# Sigmoid function
def sigmoid(x):
    return 1 / (1 + np.exp(-x))

# x and y values
x = np.linspace(-5, 5, 100)
y = sigmoid(x)

# Plot
fig = px.line(x=x, y=y)
fig.update_xaxes(title='x')
fig.update_yaxes(title='sigmoid(x)')
fig.update_layout(template="simple_white", font=dict(size=18), title_text='Sigmoid Function Plot',
                  width=650, title_x=0.5, height=400)

# Show the plot
fig.show()


Sigmoid函数
但是，出于以下原因，Sigmoid函数在前沿神经网络中并不常用：


- 
梯度消失问题——从上面的图中可以看出，在极端正或负值处的梯度接近零。这在训练神经网络时是一个问题，因为它会减慢学习和收敛速度。
- 
未以零为中心——Sigmoid的值在0和1之间，因此始终为正。这在使用基于梯度下降的算法时是一个问题，因为它会始终在一个方向上推动或下降。
- 
计算成本高——计算指数并不高效，因此这个函数在大型神经网络中尤其昂贵，因为需要计算数千次。
**Tanh**
Tanh是一种常见的三角双曲函数，将输入映射到-1和1之间。因此，它以零为中心，是对Sigmoid函数的改进。
Tanh函数的数学和视觉表示如下：


# Import packages
import numpy as np
import plotly.express as px

# tanh function
def tanh(x):
    return np.tanh(x)

# x and y values
x = np.linspace(-5, 5, 100)
y = tanh(x)

# Plot
fig = px.line(x=x, y=y)
fig.update_xaxes(title='x')
fig.update_yaxes(title='tanh(x)')
fig.update_layout(template="simple_white", font=dict(size=18), title_text='Tanh Plot',
                  width=650, title_x=0.5, height=400)

# Show the plot
fig.show()


Tanh函数
尽管以零为中心，但Tanh函数存在与Sigmoid函数类似的问题：

- 
梯度消失问题——Tanh在极端正和负值处的梯度也接近零。这在训练神经网络时是一个问题，因为它会减慢学习和收敛速度。
- 
计算成本高——Tanh需要计算更多的指数，因此计算效率甚至比Sigmoid更低。
**ReLU**
修正线性单元（ReLU）是最受欢迎的激活函数，因为它计算效率高，并解决了上述提到的梯度消失问题。
其数学和视觉表示如下：


# Import packages
import numpy as np
import plotly.express as px
import os

# relu function
def relu(x):
    return (np.maximum(0, x))

# Generate data
x = np.linspace(-5, 5, 100)
y = relu(x)

# Plot relu
fig = px.line(x=x, y=y)
fig.update_layout(template="simple_white", font=dict(size=18), title_text='ReLU',
                  width=650, title_x=0.5, height=400)
fig.show()


ReLU函数
尽管ReLU激活函数在Sigmoid和Tanh函数方面具有许多优势，但它也存在一些新问题：

- 
神经元死亡——存在“ReLU死亡问题”，即当输入值小于零时，由于这条零线，神经元对于任何输入始终输出零。
- 
值无界——梯度可以趋于正无穷。在极端情况下，如此大的数字可能会引起计算问题。
- 
未以零为中心——ReLU也不以零为中心，其均值为正。这可能会导致训练过程中的问题。
**Leaky ReLU**
我们可以调整ReLU函数以产生“Leaky”ReLU，其中负输入不是零，而是具有梯度α的浅斜率：


# Import packages
import numpy as np
import plotly.express as px

# Leaky ReLU function
def leaky_relu(x, alpha=0.01):
    return np.where(x >= 0, x, alpha * x)


# x and y values
x = np.linspace(-5, 5, 100)
y = leaky_relu(x)

# Plot
fig = px.line(x=x, y=y)
fig.update_xaxes(title='x')
fig.update_yaxes(title='Leaky ReLU(x)')
fig.update_layout(template="simple_white", font=dict(size=18), title_text='Leaky ReLU Plot',
                  width=650, title_x=0.5, height=400)

fig.show()


Leaky ReLU函数
Leaky ReLU通过具有小梯度来避免神经元死亡问题，从而使神经元可以继续学习。
更不用说它还避免了梯度消失问题，并且计算效率高，然而，与其他函数一样，它也存在一些缺陷：


- 
斜率选择：Leaky斜率的梯度没有规定，需要手动选择。不过，它始终可以通过超参数调整。
- 
未以零为中心——与普通的ReLU一样，它也不以零为中心。
- 
值无界——与ReLU一样，其梯度可以趋于正无穷，并且实际上是无界的。
**其他**
上面列出了四种最常见的激活函数，但还有其他几种，如：

- 
Swish：这是谷歌开发的ReLU函数的可微近似，swish(x) = x * sigmoid(x)。
- 
门控线性单元（GLU）：这主要用于循环神经网络。
- 
Softmax：主要用于多项式分类问题。
没有一种激活函数绝对优于另一种，最好尝试各种激活函数，看看哪种最适合你的模型。
**总结与进一步思考**
为了使神经网络满足通用逼近定理，它必须包含非线性激活函数，否则你可以将其压缩成单层，基本上就是一个线性回归模型。
有许多合适的激活函数，其中最常用的是ReLU，然而，这并不是绝对的，你应该尝试各种函数，以找到最适合你模型的函数。
