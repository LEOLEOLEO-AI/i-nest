# 海河主线年度里程碑表

生成时间：2026-05-03

## 目标

把海河实验室重大专项从“叙事性申报书”推进为“年度可执行路线”，确保每一年的工作都能对齐论文、原型和交付物。

## 总目标

- 构建 `TCC + SDI-CC` 的端侧液态硬件主线
- 形成 `3+1` 极简 MVP、多 FPGA 原型和热切换演示
- 同步支撑 `B1/B2/B3` 论文推进

## 2026：理论与标准定型

### 目标

- 完成主线口径收敛
- 固化 `SDIO-N` 基本接口与语义
- 跑通小规模原语验证

### 关键交付物

- 统一版项目申报书
- 统一版汇报材料
- `SDIO-N 1.0` 草案
- `3+1` 原语级 RTL 或模拟器原型

### 当前工作

- 统一 `TCC / SDI / 5+4 原语 / SDIO-N` 口径
- 统一申报书、指南、Marp 汇报材料与指标

### 本年要推进

- 明确 `NPC-AR / CPC-M / CPC-S / SDI-Ctrl` 的最小验证边界
- 确定 `B1/B2/B3` 的论文分工
- 建立与 `PyiNEST-Lite SDK` 的接口草案

## 2027：原型互连与生态开发

### 目标

- 搭建板卡级原型
- 跑通场景一：端侧多路 AI 视频推理 + 原生 AllReduce
- 建立 SDK 和演示基础能力

### 关键交付物

- 双/多 FPGA 板卡级原型
- 板间高速 SerDes 互连跑通
- `PyiNEST-Lite SDK` 初版
- 场景一性能报告

### 要推进

- 将 `B2` 推进到可投稿版本
- 把硬件网内 AllReduce 验证与项目指标打通
- 建立演示大屏最小版本

## 2028：跨域重构与交付验收

### 目标

- 跑通场景二：复杂雷达 DBF / FFT 热切换
- 完成一键热切换演示
- 完成交付、开源和结题展示

### 关键交付物

- 场景二验证报告
- 热切换演示系统
- 开源代码与接口文档
- 结题答辩材料与可视化大屏

### 要推进

- 把 `B3` 推进为高质量范式论文
- 将系统指标与产业化后续路线对接
- 形成定制流片或后续孵化准备包

## 当前重点

- 先完成 `2026` 阶段的统一口径和最小验证边界
- 不提前摊薄到所有远期任务
- 近期交付优先级：
  1. 统一口径
  2. 原语验证
  3. 论文分工
  4. SDK 草案

## 风险提示

- 如果 `TCC / SDI / SDIO-N` 口径不统一，后续原型、论文和答辩会彼此打架
- 如果 `3+1` MVP 边界不明确，容易陷入无上限扩展
- 如果论文与项目节奏不同步，会出现“项目有故事，论文没落稿”或“论文有框架，项目没原型”的断裂
