## **Topic**
Unified Communication-Computation Primitive Set on Software-Defined Reconfigurable Interconnect Topologies: TCC-11 Completeness, Minimality, and Sub-Microsecond Liquid Topology Switching for AI Inference, FFT, and Radar Digital Beamforming (DBF)

## **Novel Angle**

### What Has NOT Been Well-Studied

Existing work on collective communication primitives (AllReduce, AllGather, Broadcast, AlltoAll, Reduce, Scan, Map, GEMM) treats **topology as a fixed substrate** — an immutable physical constraint that algorithms must adapt to. Separately, work on reconfigurable interconnects (optical circuit switching, silicon photonics, RDMA fabrics) focuses on **network-layer reconfiguration** without exposing topology as a first-class computational primitive.

The **critical gap** is the following compound problem, which has not been addressed as a unified system:

1. **TCC-11 Completeness and Minimality as a Formal Algebraic Question**: No prior work has formally characterized whether a set of exactly 11 named communication-computation primitives (the proposed TCC-11 set) constitutes a *complete* and *minimal* basis — analogous to functional completeness in Boolean logic or a Turing-complete instruction set — over the space of distributed computation patterns arising in AI training/inference, FFT pipelines, and radar DBF. Existing collective communication libraries (TCCL, MPI, OneCCL) are designed empirically, not from a completeness-theoretic foundation.

2. **Topology-as-Computation Isomorphism**: The idea that a reconfigurable interconnect topology can be treated as an *isomorphic encoding* of a computation graph — where rewiring *is* the computation, not merely a transport mechanism — has not been formalized or empirically validated. This is distinct from prior work on in-network computing (INC) or programmable switches (P4, Tofino), which compute *within* fixed topologies rather than *through* topology mutation.

3. **Sub-Microsecond "Liquid" Topology Switching**: Achieving topology reconfiguration under **1 µs** (compatible with AI inference token latency budgets, FFT pipeline stages, and radar pulse-repetition intervals) using software-defined control of emerging optical/electrical hybrid fabrics has not been demonstrated in the context of collective primitive dispatch. Current optical circuit switches achieve ~10–100 µs reconfiguration; the sub-1-µs regime requires co-design of the control plane, primitive scheduler, and physical layer that has not been explored.

### Why This Is Timely

- **AI inference disaggregation** (prefill/decode separation, speculative decoding) creates new irregular collective patterns that fixed topologies handle poorly; reconfigurability is now a practical necessity, not a theoretical curiosity.
- **Radar DBF and MIMO signal processing** are increasingly software-defined (open RAN, cognitive radar), creating demand for the same primitive set across HPC and radar domains — a cross-domain unification opportunity that is novel.
- **Silicon photonics and MEMS-based optical switches** are reaching commercial maturity (sub-10-µs switching), making the sub-1-µs goal tractable via hybrid electrical fast-path + optical slow-path co-design.
- **Neuromorphic and near-memory compute** architectures (targeting sparse, event-driven collectives) lack a formal primitive algebra, creating a gap this work can fill.

### How This Differs from Standard Approaches

| Standard Approach | This Work |
|---|---|
| Fixed topology, optimize collective algorithms | Topology is a degree of freedom co-optimized with algorithms |
| Empirical collective library design (TCCL) | Formal completeness/minimality proof of primitive set |
| In-network compute within topology | Computation *encoded as* topology transitions |
| Single-domain (ML or HPC or radar) | Unified primitive set valid across AI, FFT, DBF |
| Microsecond-to-millisecond switching | Sub-microsecond liquid switching with primitive-aware scheduling |
