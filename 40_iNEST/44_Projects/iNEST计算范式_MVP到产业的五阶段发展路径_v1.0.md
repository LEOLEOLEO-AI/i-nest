## 二、Phase 0: 科学工具 MVP（2026 Q3 - 2027 Q2, 9个月）

### 2.1 MVP定义

> **CST-Meter v0.1**：能够在N≤512范围内准确计算CST值并检测涌现相变的最小可行产品。

**一句话卖点**：_"输入网络图，输出涌现分数——让涌现从哲学概念变成工程度量"_

### 2.2 MVP能力矩阵

| 功能模块 | 内容 | 优先级 | 负责人 |
|---------|------|--------|--------|
| **CST核心计算** | S_c, T_c, CST完整计算 | P0 | 天大 |
| **相变检测** | θ_k自动测定 + phase_label | P0 | 复旦 |
| **命令行界面** | `cst-meter --input graph.json` | P0 | 复旦 |
| **数据I/O** | JSON输入/输出 + 原始HDF5 | P1 | 天大 |
| **基础可视化** | CST-时间曲线, 相变图 | P1 | 复旦 |
| **拓扑生成器** | Ring/Butterfly/Fractal/Small-World | P1 | 天大 |
| **简单SNN引擎** | 64神经元脉冲神经网络 | P1 | 天大 |
| **雪崩基础分析** | 规模/时长分布统计 | P2 | 复旦 |
| **涌现评分** | E_score ∈ [0,1] | P1 | 复旦 |

### 2.3 MVP技术架构

```
CST-Meter v0.1
│
├── CLI: cst-meter
│   ├── cst-meter compute --input graph.json --output result.json
│   ├── cst-meter phase --input timeseries.h5 --output phase.json
│   ├── cst-meter avalanche --input spikes.h5 --output avalanche.json
│   └── cst-meter benchmark --system snn --size 256
│
├── Python API (内部使用)
│   ├── from cst_meter import CSTCalculator
│   ├── calc = CSTCalculator(graph)
│   ├── result = calc.compute()  # → {S_c, T_c, Gamma_st, CST, I_est, E_score}
│   └── phase = calc.detect_phase(cst_timeseries)
│
└── 核心算法
    ├── S_c: 图拉普拉斯特征值分布 + 谱熵
    ├── T_c: 时序多尺度熵 (MSE)
    ├── Γ_st: 空间-时间交叉互信息矩阵的SVD
    ├── CST: (S_c · T_c) · exp(α · Γ_st)
    └── θ_k: 自适应分段线性拟合 + 拐点检测
```

### 2.4 MVP成功标准

| 指标 | 目标值 | 测量方法 |
|------|--------|---------|
| CST计算精度 | R² > 0.95 (N≤512) | 与理论推导值对比 |
| θ_k检测精度 | ±10% | 与人工标注对比 |
| 计算速度 | N=512 < 60s (单GPU) | 基准测试 |
| 代码质量 | 测试覆盖率 > 80% | pytest |
| 用户验证 | 3个外部团队可用 | 内部alpha测试 |

## 四、Phase 2: 标准工具（2029 Q3 - 2031 Q2, 24个月）

### 4.1 目标：建立涌现度量的学术标准

> **Emergence-Bench v1.0**：标准化的涌现评估基准测试集，覆盖三大主流AI计算范式。

### 4.2 Emergence-Bench 完整设计

```
Emergence-Bench v1.0
│
├── EB-SNN (脉冲神经网络)
│   ├── 网络规模: 64 / 128 / 256 / 512 / 1024
│   ├── 拓扑类型: Ring / Butterfly / Fractal / SF / SW / Hybrid
│   ├── 可塑性规则: STDP / FEP / SOC / None
│   ├── 评估指标: CST, 雪崩统计, 信息论, E_score
│   └── 标准数据集: 10组×10^6 spikes per config
│
├── EB-RC (储备池计算)
│   ├── 储备池规模: 64 / 128 / 256 / 512
│   ├── 谱半径扫描: 0.1→2.0 (step 0.05)
│   ├── 输入缩放: 0.01→10.0
│   ├── 评估指标: 记忆容量, 非线性度, CST, NMAE
│   └── 标准任务: NARMA-10, Mackey-Glass, MNIST
│
├── EB-Transformer (注意力机制)
│   ├── 模型规模: 100M / 350M / 1.3B / 7B
│   ├── 架构变体: GPT-2 / LLaMA / MoE
│   ├── 注意力头数: 8 / 16 / 32 / 64
│   ├── 评估指标: CST (注意力图), 涌现能力得分, 标度律
│   └── 标准任务: MMLU, HellaSwag, BIG-Bench
│
└── EB-GNN (图神经网络)
    ├── 图类型: Erdős-Rényi / Barabási-Albert / Watts-Strogatz / Fractal
    ├── 节点数: 100 / 1000 / 10000
    ├── 评估指标: CST, 图分类准确率, 节点回归MSE
    └── 标准数据集: Cora, CiteSeer, PubMed, OGB
```

### 4.3 标准化协议

**Emergence-Bench Standard Protocol v1.0**：

```
1. 声明计算环境 (GPU型号, CUDA版本, 内存)
2. 固定随机种子 (seed=42)
3. 每个配置重复3次, 报告均值±标准差
4. 报告完整超参数
5. 提交原始数据+预处理脚本
6. 使用Emergence-Bench提供的标准评分脚本
```
